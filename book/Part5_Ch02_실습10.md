# 실습 ⑩: 나의 투자 유형 진단

Part 5의 첫 실습은 매물을 고르는 단계가 아닙니다.
그 전에 먼저, **내가 어떤 조건의 독자인지**를 확정하는 단계입니다.
이 실습에서는 AI가 8가지 질문을 순서대로 던지고, 그 답변을 바탕으로 A/B/C 타입과 핵심 입력값을 정리합니다.
여기서 확정한 결과 요약이 실습 ⑪~⑭의 출발점이 됩니다.

**학습 목표**
- AI의 8문 8답을 통해 나와 가장 가까운 A/B/C 타입을 확인합니다
- 현금, 소득, 대출, 목표 일정 같은 핵심 입력값을 한 번에 정리합니다
- 다음 실습에 바로 붙여넣을 수 있는 결과 요약 블록을 만듭니다

**핵심 키워드**
A/B/C 타입 / 투자 프로필 / 결과 요약 / 멀티턴 대화 / 가용 현금 / 합산 연소득

**예상 소요 시간**: 10~15분

이 실습은 A타입 예시로 보여드리지만, B타입과 C타입도 같은 구조로 실행할 수 있습니다.
아래 프롬프트는 그대로 두고, 샘플 투자 프로필 블록만 자신의 값으로 바꾸면 됩니다.

---

## 목적이 달라지면 보는 매물과 계산식이 달라집니다

많은 분이 실습 ⑩을 "나는 어떤 타입인가?"를 붙이는 단계로만 생각합니다.
하지만 더 중요한 역할은 따로 있습니다.
실습 ⑪ 구매력 계산, 실습 ⑫ 리스크 분석, 실습 ⑬ 타임라인, 실습 ⑭ 프로필 시트에 들어갈 **기본 입력값을 한 번에 확정하는 것**입니다.

같은 현금 3억이라도 목표가 다르면 질문 자체가 달라집니다.
월세 수익이 목적이면 임대수익률과 공실 위험을 먼저 봐야 합니다.
첫집 마련이 목적이면 교통, 생활 인프라, 대출 가능액이 더 중요합니다.
갈아타기라면 기존 집을 얼마에 팔 수 있는지와 기존 대출을 어떻게 정리할지가 핵심이고, 다운사이징이라면 대출 없이 얼마나 현금을 남길 수 있는지가 먼저입니다.

즉, 실습 ⑩은 성향 테스트가 아닙니다.
앞으로 어떤 계산을 해야 하는지, 어떤 조건을 우선순위로 둘지를 정하는 **분기점**에 가깝습니다.

아래 표처럼 같은 출발 자금이어도 목표가 바뀌면 보는 기준이 달라집니다.

| 같은 출발 조건 | 목표 | 먼저 확인할 것 |
|---------------|------|----------------|
| 현금 3억, 무주택 | 첫집 마련 | 소득, 대출, 출퇴근, 입주 시점 |
| 현금 3억, 1주택 보유 | 갈아타기 | 기존 집 매도금액, 추가 대출, 학군 |
| 현금 3억+기존 자산 | 다운사이징 | 무대출 가능 여부, 잔여 현금, 생활비 |

이 책에서 사용하는 타입은 아래 세 가지입니다.

| 타입 | 상황 | 핵심 목표 | 다음 실습에서 중요해지는 값 |
|------|------|----------|----------------------|
| **A타입** | 무주택 신혼·맞벌이 | 첫집 마련 | 가용 현금, 합산 연소득, 입주 목표 |
| **B타입** | 소형 1주택 보유 가족 | 학군지 갈아타기 | 매도 순현금, 기존 대출, 갈아타기 일정 |
| **C타입** | 서울 1주택, 은퇴 준비 | 다운사이징 + 현금흐름 | 매도 후 순현금, 무대출 여부, 생활비 구조 |

같은 "부동산 투자"라도 출발점이 다르면 계산식이 완전히 달라집니다.
A타입은 첫집 마련이 핵심이고, B타입은 갈아타기 자금 구조가 중요하며, C타입은 무대출과 현금흐름이 먼저입니다.
그래서 이 실습의 핵심은 멋진 진단 문구가 아니라, **실제 계산에 쓸 숫자와 조건을 빠짐없이 모으는 것**입니다.

이번 실습에서 AI는 8가지 질문을 하나씩 던집니다.
보유 주택 수, 가용 현금, 본인과 배우자 소득, 기존 대출, 매수 목적, 감내 가능한 손실, 관심 지역, 보유 기간을 순서대로 물어봅니다.
대화가 끝나면 타입과 함께 프로필 요약표, 그리고 다음 실습에 바로 붙여넣을 결과 요약 블록을 출력합니다.

## A/B/C 타입을 진단하고 결과 요약 블록까지 받아야 합니다

### 프롬프트를 보내기 전에

이 프롬프트는 세 가지 목적을 동시에 달성하도록 설계되어 있습니다.

첫째, **질문을 한 번에 하나씩 진행하게 만듭니다.**
실습 ⑩은 리서치형이 아니라 대화형 실습입니다.
한꺼번에 모든 질문을 던지면 독자가 중간에 빠뜨리기 쉽고, 답변 품질도 떨어집니다.

둘째, **진단 결과보다 입력값 요약을 더 중요하게 다룹니다.**
타입만 알아서는 실습 ⑪로 넘어갈 수 없습니다.
현금, 소득, 기존 대출, 관심 지역이 모두 구조화되어 있어야 다음 계산이 가능합니다.

셋째, **마지막에 결과 요약 블록을 강제로 남깁니다.**
이 블록이 있어야 다음 실습에서 복사해 붙여넣는 흐름이 자연스럽게 이어집니다.
실습 10은 독립 실습이 아니라 Part 5 전체의 입력 수집 단계이기 때문입니다.

### 실행 방법

01. ChatGPT에서 새 대화창을 엽니다.
02. 아래 프롬프트를 복사해서 붙여넣습니다.
03. AI가 질문을 하나씩 던지면 순서대로 답변합니다.
04. 마지막에 타입 진단, 프로필 요약표, 결과 요약 블록이 모두 나왔는지 확인합니다.

💡 **Tip.** B타입과 C타입도 같은 순서로 실행하면 됩니다.
다만 마지막에 붙여 넣는 샘플 투자 프로필과 한 줄 설명만 자신의 상황에 맞게 바꾸면 됩니다.

💡 **Tip.** 이 실습은 한 번에 끝나는 단문 프롬프트가 아닙니다.
첫 입력 직후에는 첫 질문만 나오는 것이 정상입니다.
8개 질문이 끝난 뒤에 최종 결과가 출력됩니다.

이 프롬프트를 실행하면, AI가 A/B/C 타입을 진단하고 다음 실습에 넘길 입력값을 표와 요약 블록으로 정리해줍니다.

### AI 응답 확인하기

독자 여러분도 직접 실행해 보세요.
여기서는 A타입 기준 결과를 먼저 보여드리겠습니다.
실제로 프롬프트를 돌려서 나온 결과와 비교하면 자기 프로필이 어떻게 달라지는지 바로 확인할 수 있습니다.
자신의 타입이 B타입이나 C타입이라면, 아래 표와 요약 블록의 값만 바꿔서 읽으면 됩니다.

A타입 기준으로 정리된 최종 결과물은 아래와 같습니다.

```markdown
타입 진단: A타입
한 줄 설명: 결혼을 앞둔 30대 초반 맞벌이 신혼부부

프로필 요약표
| 항목 | 내용 |
|------|------|
| 보유 주택 | 0채 (무주택) |
| 투자 가용 현금 | 2.3억 원 |
| 본인 연소득 | 5,800만 원 |
| 배우자 연소득 | 4,500만 원 |
| 합산 연소득 | 1.03억 원 |
| 기존 대출 | 없음 |
| 매수 목적 | 첫집 마련 |
| 리스크 허용도 | 낮음 (최대 손실 10%) |
| 관심 지역 | 수도권 |
| 타겟 면적 / 입주 목표 | 전용 59㎡ / 2026-09 |
| 예상 보유 기간 | 5년 이상 |

진단 근거:
무주택 상태에서 첫집 마련을 준비하고 있고,
가용 현금 2.3억과 합산 소득 1.03억을 기준으로
실거주 중심의 A타입 조건에 가장 가깝습니다.

다음 실습에서 중요한 값:
- 가용 현금 2.3억
- 합산 연소득 1.03억
- 기존 대출 없음
- 수도권, 전용 59㎡, 6개월 내 입주 목표
- 보유 기간 5년 이상
```

현재 검토 메모 기준으로 A타입에서 확인된 핵심 값은 아래와 같습니다.

| 항목 | 확인된 값 |
|------|----------|
| 가용 현금 | **2.3억** |
| 합산 연소득 | **1.03억** |
| 타겟 면적 | **59㎡** |
| 타입 | **A타입** |
| 입주 목표 | **2026-09** |

이 다섯 항목은 실습 ⑪ 구매력 계산의 핵심 입력값이 됩니다.
실습 ⑩의 목적은 결국 이 숫자를 정확히 받아내는 데 있습니다.

실무적으로는 아래 결과 요약 블록이 가장 중요합니다.
다음 실습에서는 이 블록을 그대로 붙여넣으면 됩니다.

```text
--- 실습⑩ 결과 요약 ---
타입: A타입
보유주택: 0채 (무주택)
가용현금: 2.3억
본인연소득: 5,800만
배우자연소득: 4,500만
합산연소득: 1.03억
기존대출: 없음
매수목적: 첫집 마련
리스크허용도: 최대 손실 10%
관심지역: 수도권
타겟면적: 전용 59㎡
입주목표: 2026-09
보유기간: 5년 이상

```

### 확인할 포인트

| 확인 항목 | 기준 | 문제 시 조치 |
|----------|------|-------------|
| 질문을 하나씩 진행하는가 | 8문 8답 흐름 유지 | "질문을 한 번에 하나씩 진행해주세요" |
| 마지막에 프로필 요약표가 나오는가 | 표 1개로 핵심 입력값 정리 | "프로필 요약표를 표로 다시 정리해주세요" |
| 결과 요약 블록이 나오는가 | 다음 실습에 복붙 가능해야 함 | "실습⑩ 결과 요약 블록을 출력해주세요" |
| 숫자를 임의 추정하지 않는가 | 모르면 미정으로 표시 | "미입력 항목은 추정하지 말고 표시해주세요" |

> ⚠️ **주의:** 실제 채팅에서는 첫 질문부터 시작하는 것이 정상입니다.
> 다만 책 본문에서는 질문 흐름보다 최종 결과물이 더 중요하므로, A타입 기준 결과물을 먼저 확인하는 방식으로 보는 편이 효율적입니다.

## 결과 요약 블록을 저장하고 실습 ⑪로 넘어갑니다

실습 ⑩이 끝나면 가장 먼저 할 일은 **결과 요약 블록 저장**입니다.
실습 ⑪은 이 블록을 붙여넣는 순간부터 계산이 쉬워집니다.
반대로 여기서 값을 흘리면 뒤 실습이 모두 다시 확인 단계로 돌아갑니다.

저장할 핵심 항목은 아래 여섯 가지입니다.

```text
타입
가용현금
합산연소득
기존대출
관심지역
타겟면적 / 입주목표
```

| 저장 항목 | 다음 실습에서 쓰는 이유 |
|----------|---------------------|
| 타입 | A/B/C 조건별 계산 분기 |
| 가용 현금 | LTV 역산, 구매력 계산 |
| 합산 연소득 | DTI / DSR 계산 |
| 기존 대출 | 실제 대출 가능액 계산 |
| 관심 지역 | 규제지역 / 비규제지역 판단 |
| 타겟면적·입주목표 | Part 6 조건 필터와 일정 연결 |
| 보유 기간 | 리스크 분석과 타임라인 설정 |

💾 **인계 정보** — 타입(A/B/C), 가용현금, 합산연소득, 기존대출, 관심지역, 타겟면적, 입주목표, 보유기간

---

## 핵심 정리

- **실습 ⑩의 핵심은 입력값 확정입니다**: 타입 라벨보다 더 중요한 것은 현금, 소득, 대출, 지역, 일정 같은 다음 실습용 숫자와 조건입니다.
- **멀티턴 구조가 정상입니다**: 첫 입력 직후에는 첫 질문만 나오는 것이 맞고, 8개 질문이 끝난 뒤에 최종 결과가 정리됩니다.
- **결과 요약 블록을 반드시 저장합니다**: 이 블록이 실습 ⑪ 구매력 계산의 시작점입니다.
