# 실습 ⑬: 투자 타임라인 설정

실습 ⑫에서 조건부 적합 판정이 나왔다면, 이제 남은 질문은 하나입니다.
좋은 조건을 안다고 해서, 실제로 언제 움직일 것인가?
이번 실습에서는 A타입 기준으로 6개월 내 입주를 목표로 전체 로드맵을 먼저 잡고, 이어서 월별 상세 일정과 비용 타임라인까지 구체화합니다.
실습 ⑭ 프로필 시트는 여기서 나온 마일스톤과 총비용예상을 그대로 이어받습니다.

**학습 목표**
- 대출승인부터 입주까지 핵심 마일스톤을 설정합니다
- 월별 일정, 비용 발생 시점, 리스크 버퍼를 구체화합니다
- 다음 실습에 넘길 일정 요약 블록을 완성합니다

**핵심 키워드**
마일스톤 / 버퍼 / 계약금 / 잔금 / 입주 목표 / 비용 타임라인

---

## 다짐과 계획은 다릅니다

'올해 안에 집을 사야지'는 다짐입니다.
'4월 대출승인, 6월 매물확정, 8월 잔금, 9월 입주'는 계획입니다.

부동산 매수는 한 단계가 밀리면 뒤 일정이 전부 밀립니다.
그래서 A타입처럼 6개월 내 입주가 중요한 경우, 일정은 취향이 아니라 리스크 관리 수단입니다.

| 구분 | 다짐 | 계획 |
|------|------|------|
| 대출 | 나중에 알아본다 | 4월 안에 승인 구조 확정 |
| 매물 | 좋은 거 나오면 본다 | 5~6월 집중 임장 |
| 계약 | 맞는 집 있으면 한다 | 6월 말 전후 계약 |
| 입주 | 되면 빨리 들어간다 | 9월 2주 전입 완료 |

---

## 6개월 로드맵을 먼저 잡습니다

### 프롬프트를 보내기 전에

지금 기준일은 **2026-03-19**입니다.
이 날짜는 실습 작성 시점 기준이므로, 독자는 자신의 오늘 날짜로 바꿔서 프롬프트에 입력하면 됩니다.
A타입의 입주 목표는 6개월 내이므로, 늦어도 **2026년 9월 초 전입 완료**를 목표로 두는 일정이 필요합니다.

이번 턴의 목적은 전체 뼈대를 만드는 것입니다.
대출 승인, 지역 확정, 매물 확정, 잔금, 입주 순서를 실제 달력 위에 올려놓습니다.

### 실행 방법

01. 새 대화창을 엽니다.
02. 실습 ⑫ 결과 요약을 먼저 붙여넣습니다.
03. 아래 프롬프트를 입력합니다.

### 🤖 프롬프트 (Type A - 직접 실행형)

```markdown
[Role]
당신은 부동산 투자 일정 관리 전문가입니다.

[Mission]
위 결과 요약을 바탕으로 6개월 내 입주 완료 로드맵을 설계해주세요.

[Output Format]
1. 핵심 마일스톤
2. 단계별 기간 배분
3. 일정 해석

[Rules]
- 오늘 날짜 기준 실제 달력 날짜를 사용하세요
- 각 단계에 버퍼를 포함하세요
- 발표되지 않은 정책을 창작하지 마세요
```

### AI 응답 확인하기

A타입 예시 로드맵은 아래와 같습니다.

| 마일스톤 | 목표 시기 | 완료 기준 |
|----------|----------|----------|
| 대출 사전승인 | **4월 2주** | 승인서 수령 |
| 지역 확정 | **5월 4주** | 3곳 -> 1곳 선정 |
| 매물 확정 | **6월 4주** | 계약 체결 |
| 잔금 + 등기 | **8월 4주** | 등기부등본 확인 |
| 입주 | **9월 2주** | 전입신고 완료 |

| 단계 | 기간 | 버퍼 |
|------|------|------|
| 리서치 + 대출심사 | **3주** | **+1주** |
| 매물 탐색 + 임장 | **6주** | **+2주** |
| 계약 + 잔금 | **6주** | **+2주** |
| 입주 | **2주** | **+1주** |

이 일정의 핵심은 4월 안에 대출 구조 확정, 6월 안에 매물 확정, 8월 안에 잔금 처리입니다.
버퍼를 빼면 일정이 예뻐 보이지만, 실제로는 승인 지연과 매물 재탐색 때문에 실패할 가능성이 커집니다.

### 확인할 포인트

| 확인 항목 | 기준 | 문제 시 조치 |
|----------|------|-------------|
| 기준일이 실제 날짜인가 | 2026-03-19 기준 | "오늘 날짜 기준으로 다시 잡아줘" |
| 마일스톤이 5개로 정리되는가 | 승인/지역/매물/잔금/입주 | "핵심 단계를 5개로 다시 정리해줘" |
| 버퍼가 포함되는가 | 각 단계에 +1~2주 | "버퍼를 포함해서 다시 계산해줘" |

## 월별 일정과 비용을 실제 행동으로 바꿉니다

### 프롬프트를 보내기 전에

로드맵은 방향이고, 월별 일정은 실행입니다.
이번 턴에서는 '언제 무엇을 하고', '언제 돈이 나가고', '어디서 일정이 밀릴 수 있는지'를 한눈에 보이게 만듭니다.

실습 ⑫ 리스크 분석 결과를 반영하여 추천가격대 하단인 **5.2억**을 기준 매입가로 잡았습니다. A타입 예시에서는 이 5.2억을 기준으로 계약금 10%와 취득세·중개보수·법무사 비용까지 계산해 총비용예상을 산출했습니다.

### 실행 방법

01. 같은 대화창에서 계속 진행합니다.
02. 아래 프롬프트를 붙여넣습니다.

### ➡️ 후속 프롬프트 (Type D)

```markdown
위 로드맵을 월별 상세 일정으로 전개해주세요.

1. 월별 투자 타임라인
2. 비용 타임라인
3. 리스크 버퍼

마지막에 마일스톤, 총비용예상, 핵심 일정 리스크를 포함한 결과 요약 블록을 출력하세요.
```

### AI 응답 확인하기

A타입 예시 결과에서 확정된 월별 타임라인은 아래와 같습니다.

| 월 | 주요 활동 | 비용 발생 |
|----|----------|----------|
| 1개월차 | 대출 사전심사 + 지역 1차 스크리닝 | **0~10만원** |
| 2개월차 | 지역 압축 + 임장 시작 | **20~40만원** |
| 3개월차 | 지역 확정 + 매물 집중 탐색 | **20~30만원** |
| 4개월차 | 매물 확정 + 계약 체결 | **5,200만원** |
| 5개월차 | 잔금 준비 + 입주 준비 | **30~80만원** |
| 6개월차 | 잔금 + 등기 + 이사 + 입주 마무리 | **약 24,050만원 + 이사비** |

총비용예상은 **약 5억 2,980만원**으로 정리됩니다.
다만 이 값은 매매가 전체 흐름을 보여주는 표이고, 실제 자기자본 집행은 여기서 정책대출 실행분을 빼서 봐야 합니다.

리스크 버퍼도 같이 정리되었습니다.

| 구간 | 리스크 | 버퍼 | 대안 |
|------|--------|------|------|
| 대출 승인 | 승인 지연 | **+1주** | 정책대출·일반주담대 병행 상담 |
| 매물 탐색 | 적합 매물 부재 | **+2주** | 후보 지역 1곳 추가 확보 |
| 잔금~입주 | 일정 공백 | **+1주** | 잔금일 조정 특약, 입주 준비 선확보 |

특히 잔금~입주 구간이 4주 이상 벌어지면 이사·보관·단기거주 비용이 한꺼번에 커지므로, 이건 계약 전에 반드시 확인해야 하는 핵심 리스크로 봐야 합니다.

### 확인할 포인트

| 확인 항목 | 기준 | 문제 시 조치 |
|----------|------|-------------|
| 계약금과 총비용예상이 계산되는가 | 5.2억 기준 계산 | "계약금과 총비용을 다시 계산해줘" |
| 잔금~입주 공백이 점검되는가 | 4주 이상이면 리스크 | "잔금~입주 공백 리스크도 넣어줘" |
| 결과 요약 블록이 있는가 | 다음 실습용 | "마지막 요약 블록을 출력해줘" |

## 핵심 정리

- **일정은 리스크 관리입니다**: 6개월 내 입주 목표라면 날짜를 고정해야 뒤 계산이 실제 행동으로 이어집니다.
- **A타입의 핵심 마일스톤은 4월 승인, 6월 매물확정, 8월 잔금, 9월 입주입니다**: 이 흐름이 무너지면 정책대출과 입주 목표 둘 다 흔들립니다.
- **총비용예상은 약 5억 2,980만원입니다**: 매매가 전체 흐름을 보여주는 숫자이며, 실제 자기자본 집행은 정책대출 실행분을 제외해 봐야 합니다.
- **다음 실습 연결**: 이 일정과 비용 요약을 실습 ⑭에 넣어 최종 투자 프로필 시트로 종합합니다.
