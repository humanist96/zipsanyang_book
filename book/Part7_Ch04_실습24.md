# 실습 ㉔: 매물 검토 — 실거래가와 호가로 협상 근거 만들기

실습 ㉒에서 민락푸르지오를 1순위 단지로 선정하고, 실습 ㉓에서 5.4~6.2억이 적정가라는 것까지 확인했습니다.
이제 네이버부동산을 열면 같은 단지, 같은 면적인데 가격이 제각각입니다.
이번 실습에서는 실거래가와 호가를 직접 비교 분석하여, 데이터에 근거한 협상 목표가를 산출합니다.
2단계 프롬프트 구조를 활용하여 AI가 데이터를 수집하고 종합 판단을 내리는 과정을 진행합니다.
겉으로는 숫자 비교 실습처럼 보이지만, 실제 핵심은 "없는 숫자를 만들지 않기"입니다. 최근 데이터가 충분하지 않으면 협상가를 억지로 정하지 않고 멈추는 것이 더 좋은 판단일 수 있습니다.
많은 분이 이 단계에서 숫자가 갑자기 많아져 어렵게 느낍니다. 하지만 실제로는 "최근에 얼마에 팔렸고, 지금 얼마에 내놨는가"를 같은 표에 올려놓는 작업이라고 생각하면 됩니다.

**학습 목표**

- 실거래가와 호가의 차이를 이해하고 협상 전략에 활용합니다
- 2단계 프롬프트 구조로 AI에게 정확한 분석을 요청합니다
- 거래량 추이와 시나리오 분석으로 매수 타이밍을 판단합니다

**핵심 키워드**
실거래가 / 호가 / 호가-실거래 차이율 / 2단계 프롬프트 / 매수 실행 전략 / 거래량 추이 / 시나리오 분석

## 같은 단지인데 가격이 왜 다른가

실습 ㉒에서 선정한 민락푸르지오 84㎡를 네이버부동산에서 검색하면 5억 5천, 5억 8천, 6억 2천처럼 가격이 제각각입니다.
같은 단지, 같은 면적인데도 층수, 향, 동, 옵션 같은 세부 조건이 다르기 때문입니다.
같은 단지 안에서도 3층과 20층은 억 단위로 차이 날 수 있고, 남향과 북향은 수천만 원 차이가 날 수 있습니다.

가격 차이보다 더 중요한 것이 있습니다.
바로 **실거래가**와 **호가**의 차이입니다.

**실거래가** — 계약서에 도장 찍힌 실제 거래 가격입니다. 국토부에 신고되어 공개되므로 누구나 아실(asil.kr)에서 확인할 수 있습니다.

**호가** — 매도자가 "이 가격에 팔겠다"고 올려놓은 가격입니다. 보통 실거래가보다 높습니다.

| 구분               | 정의                 | 특징                         |
| ------------------ | -------------------- | ---------------------------- |
| **실거래가** | 실제로 거래된 가격   | "진짜 시세" — 공식 데이터   |
| **호가**     | 매도자가 부르는 가격 | "희망 시세" — 협상의 시작점 |

핵심은 **호가가 실거래가보다 얼마나 높은가**입니다.
이 차이율에 따라 협상 전략이 달라집니다.
호가는 협상의 출발선이고, 실거래가는 시장이 실제로 찍은 점수표에 가깝습니다.

| 호가-실거래 차이율 | 의미                    | 협상 전략                   |
| ------------------ | ----------------------- | --------------------------- |
| **15% 이상** | 매도자가 너무 높게 부름 | 협상 여지 큼, 급매 탐색     |
| **5~15%**    | 적당한 협상 여지        | 10% 정도 깎기 시도          |
| **5% 이하**  | 시장 균형 상태          | 깎기 어려움, 빠른 결정 필요 |

차이율을 알면 **데이터 기반 협상**이 가능합니다.
"최근 실거래가 5.6억인데요, 5.5억에 해주실 수 있어요?"처럼 근거를 가지고 협상할 수 있습니다.
막연히 "좀 깎아주세요"라고 말하는 것과는 설득력이 완전히 다릅니다.

## 실거래가와 호가 비교 분석하기

이 실습은 **2단계 프롬프트 구조**로 진행합니다.
1단계에서 AI에게 역할과 분석 규칙을 부여하고, 2단계에서 분석 대상 단지 정보를 입력합니다.
두 단계를 분리하는 이유는 정확도 때문입니다. 한 번에 모든 정보를 입력하면 AI가 검증 없이 분석을 시작할 수 있습니다. 2단계 구조는 입력이 완료되기 전까지 분석을 진행하지 않도록 강제합니다.

### 프롬프트를 보내기 전에

데이터 소스를 먼저 확인해야 합니다.
실거래가는 아실(asil.kr), 호가는 네이버부동산(land.naver.com)을 사용합니다.
아실을 사용하는 이유는 국토부 실거래가 데이터를 층별, 동별로 정리해서 제공하기 때문입니다.
분석 전에 두 사이트에서 분석할 단지의 기본 정보를 직접 확인해 두면 AI 응답과 교차검증할 수 있습니다.
만약 AI가 최근 데이터를 충분히 못 찾으면, 사용자가 직접 확인한 최근 실거래 1~3건만 붙여넣어도 정확도가 크게 올라갑니다. 웹 검색 기능보다 이 한두 줄의 직접 확인 데이터가 더 강할 때도 많습니다.

### 실행 방법

1. ChatGPT에서 새 대화창(New Chat)을 엽니다.
2. 웹 검색 기능을 켤 수 있으면 켭니다.
3. 아래 **1단계 고정 프롬프트**를 복사해서 붙여넣습니다.
4. AI가 "2단계 사용자 입력 폼"을 출력하면, 폼에 분석 대상 단지 정보를 입력합니다.
5. AI가 실거래가와 호가를 수집하여 비교 분석 결과를 표 형식으로 출력합니다.

💡 **Tip.** 최신모델의 웹 검색 기능을 켜면 최신 실거래 데이터를 직접 조회합니다. 웹 검색 없이 진행하면 AI가 학습 데이터 기반으로 응답하여 최신 수치와 차이가 날 수 있습니다.

```
[Critical Rules: 매수 실행 전략 판정]

4) 다른 단지 추천: 해당 단지의 고평가가 심각하거나 리스크가 커서 인근의 저평가된 대체 단지로 유도해야 하는 경우.

지금부터 2단계 사용자 입력 폼을 출력하세요.
```

### 프롬프트 — 2단계: 사용자 입력 폼 (단지 정보 입력)

1단계 프롬프트를 전송하면 AI가 아래 형식의 입력 폼을 출력합니다.
폼에 분석할 단지 정보를 입력하여 제출하면 분석이 시작됩니다. 폼에 조건을 입력할 때는 대괄호(`[ ]`) 안의 내용만 본인 조건으로 교체하면 됩니다.

```
[사용자 입력 폼]

| 항목 | 입력값 |
|------|--------|
| 분석 기준일 | [YYYY-MM-DD] |
| 단지명 | [아파트 단지명] |
| 지역(시/구/동) | [시/구/동] |
| 면적 입력(아래 중 최소 1개는 필수입니다) |  |
| - 전용면적(㎡) | [예: 59 / 84 / 59㎡ / 84㎡ / 모름] |
| - 평형(평, 통상 공급면적 기준) | [예: 24평 / 34평 / 모름] |
| 추가 면적(선택) | [전용㎡ 또는 평형(평) 또는 없음] |
| 내가 직접 확인한 최근 실거래(선택) | [예: 2026.01 / 12층 / 5.6억 / 101동, 없으면 없음] |
| 내가 직접 확인한 현재 호가(선택) | [예: 5.8억 / 12층 / 남동향 / 101동, 없으면 없음] |

면적 입력 안내(참고)
- "59/84"처럼 숫자만 입력하시면 기본적으로 전용㎡로 간주됩니다.
- "24평/34평"처럼 평으로 입력하시면 기본적으로 평형(공급면적 기준)으로 간주됩니다.
- 평형만 입력하셨고 전용㎡ 매핑이 불명확하면, 분석 전에 1~2개 확인 질문이 있을 수 있습니다.
```

입력 예시(의정부 민락푸르지오의 경우):

```
분석 기준일: 2026-01-05
단지명: 민락푸르지오
지역: 경기도 의정부시 민락동
전용면적: 84㎡ (30평)
내가 직접 확인한 최근 실거래: 2025.12 / 12층 / 5.6억 / 101동
내가 직접 확인한 현재 호가: 2026.01 / 5.8억 / 12층 / 남동향 / 101동
```

입력이 간단한 것이 이 프롬프트의 강점입니다.
단지명과 면적만 입력해도 되지만, 내가 직접 확인한 최근 실거래나 호가를 함께 넣으면 정확도가 훨씬 높아집니다. 특히 웹 검증이 불안정한 앱에서는 이 한두 줄의 직접 확인 데이터가 결과 품질을 크게 좌우합니다.

### AI 응답 확인하기

의정부 민락동 민락푸르지오 84㎡를 기준으로 분석한 경우 다음과 같은 결과가 나옵니다.

**실거래가 현황** (아실 기준)

| 계약일     | 가격(억) | 층수 | 동    | 비고 |
| ---------- | -------- | ---- | ----- | ---- |
| 2025.12.18 | 5.6      | 12층 | 101동 | -    |
| 2025.12.05 | 5.5      | 8층  | 102동 | -    |
| 2025.11.27 | 5.7      | 15층 | 101동 | -    |
| 2025.11.15 | 5.8      | 20층 | 103동 | 고층 |
| 2025.11.08 | 5.4      | 3층  | 102동 | 저층 |

이 표에서 중요한 것은 층수와 가격의 관계입니다.
3층은 5.4억이고, 20층은 5.8억입니다.
고층일수록 나중에 팔 때도 유리합니다.

**호가 현황** (네이버부동산 기준)

| 동    | 호가(억) | 층수 | 방향   | 옵션   |
| ----- | -------- | ---- | ------ | ------ |
| 101동 | 5.5      | 5층  | 남향   | -      |
| 101동 | 5.8      | 12층 | 남동향 | 풀옵션 |
| 102동 | 6.0      | 18층 | 남향   | -      |
| 103동 | 6.2      | 22층 | 남동향 | 확장   |
| 102동 | 5.9      | 10층 | 동향   | -      |

**비교 분석표**

| 항목                       | 값                                     |
| -------------------------- | -------------------------------------- |
| 실거래가 평균 (최근 3개월) | 5.6억                                  |
| 호가 평균                  | 5.88억                                 |
| 호가-실거래 차이율         | **+5.0%**                        |
| 해석                       | 적당한 협상 여지, 5~10% 깎기 시도 가능 |

차이율 5%는 약 3천만 원에 해당합니다.
5.8억 호가 매물이라면 5.5~5.6억까지 협상해볼 수 있다는 의미입니다.

**거래량 추이**

| 월      | 거래량 | 평균가 | 추세    |
| ------- | ------ | ------ | ------- |
| 2025.12 | 2건    | 5.55억 | ↓ 감소 |
| 2025.11 | 3건    | 5.63억 | → 보합 |
| 2025.10 | 4건    | 5.50억 | -       |

거래량이 줄고 있습니다.
12월에 2건은 거래가 거의 없는 수준입니다.
거래량 감소에 가격이 횡보하면 **관망 국면**으로 해석합니다.
급하지 않다면 조금 더 지켜봐도 됩니다.

**시나리오 분석** (6개월 후 예상)

| 시나리오       | 6개월 후 예상가 | 조건                         |
| -------------- | --------------- | ---------------------------- |
| **기본** | 5.5~5.8억       | 현재 추세 유지               |
| **상향** | 6.0~6.3억       | GTX-C 착공 + 금리 인하       |
| **하향** | 5.2~5.5억       | 거래량 추가 감소 + 금리 동결 |

기본 시나리오가 가장 확률이 높습니다.
GTX-C 호재가 구체화되면 상향 시나리오가 될 수 있지만, 보수적으로 5.5~5.8억을 기준으로 판단하는 것이 안전합니다.

> ⚠️ **주의:** AI가 응답한 실거래가와 호가 수치는 반드시 아실과 네이버부동산에서 직접 교차검증해야 합니다. 호가 매물은 시시각각 변하므로, AI가 분석한 시점의 매물이 이미 거래되었을 수 있습니다.

## 분석 결과로 매물 고르기

AI 분석이 완료되면 실제 매물을 선택할 기준이 생깁니다.

중고층(10~20층), 남향 또는 남동향, 호가가 실거래가 대비 5% 이내인 매물이 기본 조건입니다.
저층(1~3층)과 북향은 나중에 팔 때 유동성이 떨어집니다.
호가가 실거래가 대비 15% 이상인 매물은 매도자가 희망가를 과도하게 높인 경우입니다.

> ❓ **궁금해요 급매란?**
>
> 급매는 시세보다 10% 이상 저렴하게 나온 매물을 말합니다.
> 급매는 가격이 매력적이지만, 하자나 법적 문제가 있는 경우도 있습니다.
> 급매를 발견했다면 반드시 등기부등본과 임대차 현황을 먼저 확인해야 합니다.

분석 결과를 기반으로 최종 의사결정을 다음과 같이 정리할 수 있습니다.

| 항목                       | 민락푸르지오 분석 결과  |
| -------------------------- | ----------------------- |
| **타겟 매물**        | 5.8억 / 12층 / 남동향   |
| **실거래가 평균**    | 5.6억 (2025.10~12 기준) |
| **호가-실거래 차이** | +5.0%                   |
| **협상 목표가**      | 5.5억                   |
| **매수 실행 전략**   | 가격 협상               |

💡 **Tip.** "최근 실거래가 5.6억인데요, 5.5억에 해주실 수 있어요?"는 근거 있는 협상입니다. 막연히 "좀 깎아주세요"라고 말하는 것과는 설득력이 완전히 다릅니다. AI가 산출한 실거래가 데이터가 협상 테이블에서 가장 강력한 근거입니다.

## 핵심 정리

- **실거래가 vs 호가**: 실거래가는 계약서에 찍힌 진짜 시세이고, 호가는 매도자의 희망가입니다. 반드시 구분하여 분석합니다.
- **호가-실거래 차이율**: 차이율 15% 이상이면 협상 여지가 크고, 5% 이하이면 빠른 결정이 필요합니다.
- **2단계 프롬프트**: 고정 프롬프트로 규칙을 먼저 설정하고, 입력 폼으로 단지 정보를 제출하면 AI가 정확하게 분석합니다.
- **데이터 기반 협상**: 실거래가 수치를 근거로 제시하면 협상력이 높아집니다.
