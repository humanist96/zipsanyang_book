# 실습 ㉒: 단지 심층 비교 분석

Part 6에서 유망 지역을 선정했다면, 이제는 그 지역 안에서 어떤 단지를 선택할 것인지 결정할 차례입니다.
지역 선정이 거시적 판단이라면, 단지 선정은 상품성·입지 세부·수급·가격·리스크 등 미시적 요소를 종합하는 작업입니다.
이 실습에서는 2단계 구조화 프롬프트를 활용해 후보 단지 2~3개를 체계적으로 비교하고, 최적 단지 1개를 선정합니다.

**학습 목표**

- 5가지 평가지표(상품성·입지·수급·가격밸류·리스크)로 후보 단지를 체계적으로 비교합니다
- Self-Consistency 기법으로 AI 분석의 신뢰도를 검증합니다
- 보수·중립·공격 3가지 시나리오 분석을 활용해 나의 리스크 성향에 맞는 단지를 선정합니다

**핵심 키워드**
5가지 평가지표 / Self-Consistency / 시나리오 분석 / 가중치 채점 / 교차 검증 / AI 분석 신뢰도

---

## 지역 선정과 단지 선정은 다른 작업이다

의정부 민락동을 1순위 지역으로 선정했다고 가정해 보겠습니다.
예산 6억, GTX-C 접근성, 신축 30평대라는 조건에 맞는 지역이었습니다.
그런데 막상 네이버 부동산을 열어보면 민락동에만 비슷한 조건의 단지가 여러 개입니다.
민락푸르지오, 민락센트럴자이, 민락휴먼시아……

'다 비슷해 보이는데, 뭘 기준으로 골라야 하지?'

지역을 고르는 것과 단지를 고르는 것은 완전히 다른 작업입니다.

```
Part 6: 지역 선정 → "어느 동네?" (거시적 요소: 교통, 개발호재, 인구 유입)
    ↓
Part 7: 단지 선정 → "어느 아파트?" (미시적 요소: 상품성, 입지 세부, 수급, 가격, 리스크)
```

지역은 교통·개발호재·인구 유입 같은 거시적 요소로 판단합니다.
반면 같은 지역 안에서 단지를 고를 때는 상품성, 입지의 세부 차이, 수급 구조, 가격 밸류, 리스크 같은 미시적 요소를 분석해야 합니다.
이 미시적 요소를 체계적으로 비교하는 것이 실습 ㉒의 핵심입니다.

---

## 5가지 평가지표로 단지를 채점하는 방법

단지를 비교할 때는 다음 5가지 관점에서 종합적으로 평가합니다.
각 지표에는 가중치가 부여되어 있어, 더 중요한 요소가 전체 점수에 더 큰 영향을 미치도록 설계되었습니다.

| 평가지표             | 가중치 | 평가 내용                                     |
| -------------------- | ------ | --------------------------------------------- |
| **상품성**     | 20%    | 스펙, 브랜드, 설계, 주차, 커뮤니티, 관리/하자 |
| **입지**       | 30%    | 교통, 직주근접, 학군, 상권, 환경, 혐오시설    |
| **수급**       | 20%    | 실수요, 투자수요, 입주물량, 전세수급          |
| **가격·밸류** | 15%    | 동급지 대비 프리미엄, 거래 분포, 가격방어력   |
| **리스크**     | 15%    | 정책/금리, 규제, 공급쇼크, 구축, 유동성       |

입지가 30%로 가장 높은 비중을 차지하는 이유는 명확합니다.
부동산의 가치는 결국 위치에서 나오기 때문입니다.
같은 동네라도 역에서 5분 거리인 단지와 15분 거리인 단지는 억 단위로 시세가 차이 날 수 있습니다.

> ⚠️ **주의:** 리스크 점수는 역으로 계산합니다. 리스크가 적을수록 점수가 높고(5점), 리스크가 클수록 점수가 낮습니다(1점). "리스크 점수가 높다"는 것은 "안전하다"는 의미입니다. AI 결과를 해석할 때 반드시 기억해야 할 부분입니다.

## 1단계 고정 프롬프트로 분석 환경 설정하기

이 실습의 프롬프트는 2단계로 구성됩니다.
1단계는 AI의 운영 규칙·출력 형식·검증 로직을 설정하는 고정 프롬프트이고, 2단계는 분석 대상 단지와 내 조건을 입력하는 사용자 입력 폼입니다.
1단계를 먼저 보내 분석 환경을 구성한 다음, 2단계 폼을 채워 제출하면 됩니다.

### 프롬프트를 보내기 전에

1단계 프롬프트는 AI에게 분석 방식과 출력 형식을 미리 지정하는 역할을 합니다.
한 번만 설정해두면 같은 대화 세션에서 단지 조건을 바꿔가며 반복 분석이 가능합니다.

💡 **Tip.** 이 프롬프트는 ChatGPT의 최신모델에서 웹 검색 기능을 켜고 사용할 때 가장 효과적입니다. 실거래가, 입주물량 등 최신 데이터를 AI가 직접 조회합니다.

### 실행 방법

1. ChatGPT에서 새 대화창(New Chat)을 엽니다.
2. 최신모델을 선택하고, 웹 검색 기능을 켭니다.
3. 아래 1단계 프롬프트 전체를 복사해서 붙여넣습니다.
4. AI가 "사용자 입력 폼"을 출력하면 2단계로 넘어갑니다.

### 프롬프트

```
[ROLE]
당신은 한국 아파트 단지 분석 전문가이자 투자 리스크 매니저입니다.
근거는 길게(항목당 4~8줄) 작성하되, 내부 추론(사고 과정)은 장황하게 공개하지 마세요.
불확실하면 점수 범위와 신뢰도(High/Med/Low)를 함께 제시하세요.

[2-STEP WORKFLOW | 매우 중요]
- 본 대화는 2단계로 진행됩니다.
  1) (당신) "사용자 입력 폼"을 출력하고, 사용자가 폼을 채워 제출하도록 안내하세요.
  2) (사용자) 폼을 제출하면, 당신은 입력 검증을 통과한 경우에만 웹 수집/표준화 1회 → Run A/B/C 분석을 수행하세요.

[절대 규칙: 입력 완료 전 분석 금지]
- 사용자의 입력이 '필수 항목'을 모두 충족하기 전에는 웹 수집/분석을 시작하지 마세요.
- 누락/모순이 있으면 분석 대신, 최대 6개 이내의 짧은 질문으로 보완 입력을 요청하세요.
- 질문이 6개를 초과할 것 같으면, "가장 중요한 3개"만 먼저 질문하고 나머지는 다음 턴으로 넘기세요.

[필수 항목(모두 필수)]
- 분석 기준일
- 투자 목적(시세차익/월세수익/실거주)
- 보유 기간(N년)
- 실거주 계획(가능/불가능)
- 자금(자기자본 X억 / 대출여력 Y억 / 월상환한도 Z만원)
- 규제·세금 전제(1주택/다주택, 규제/비규제지역)
- 리스크 성향(보수/중립/공격)
- 세대수 가드레일(최소 세대수, 강도=필수/선호/무관, 예외 허용=예/아니오)
- 단지명 리스트(2~3개)
- 지역 힌트(시/구/동)
- 관심 전용면적/평형

[목표]
사용자가 제공한 단지명 리스트(2~3개)를 입력으로 받아,
1) 웹 검색으로 단지 메타데이터 및 최근 실거래·전세를 자동 수집해 표준화하고,
2) 항목별 근거를 충분히 제시하며 비교하며,
3) 시나리오(보수/중립/공격)를 포함해 최종 선택 1개(또는 조건부/보류/대안)를 제시하세요.

[평가지표(0~5) 및 가중치]
상품성: 20
입지: 30
수급: 20
가격·밸류: 15
리스크: 15 (안전할수록 높게 평가하세요)

[세대수(규모)·유동성 리스크 보정 룰: B+C 혼합(강제)]
- 세대수는 '가점 요소'가 아니라 '리스크/신뢰도 보정 요소'로만 사용하세요.
- 가격·밸류 점수에는 세대수/규모를 반영하지 마세요.

B) 절대(Absolute) 소단지 패널티
- 300세대 이상: 0
- 200~299: -0.5
- 150~199: -1.0
- 150 미만: -1.5

C) 지역 퍼센타일 패널티
- 비교군(동/생활권 1~2km + 유사 준공 ±7년 + 유사 주력면적 ±10㎡)이 10개 이상일 때만 적용하세요.
- 하위 10%: -0.5 / 하위 10~20%: -0.25 / 그 외 0
- 상위 퍼센타일에 대해서는 가점을 부여하지 마세요(대단지 편향 방지).

- 최종 규모 패널티 = B + C (하한 캡: 최소 -1.5)
- 반영 위치: 수급/리스크에만 감점으로 반영하세요. 상품성/입지/가격·밸류에는 직접 반영하지 마세요.
- 유동성 경고: 주력면적 기준 최근 12개월 거래가 연 3건 이하이면,
  * 수급 점수 상한 3점, 리스크 점수 상한 2점, 신뢰도는 Med 이하로 제한하세요.

[하드 컷(최종선택 제한)]
- 강도=필수 또는 예외 허용=아니오인 경우,
  세대수 < 최소 세대수 기준 단지는 최종 선택 후보에서 제외하세요(비교표에는 포함).
- 최종 선택은 '세대수 기준 통과' 단지 중에서만 가능하도록 하세요.
  통과 단지가 없으면 무조건 '보류' 또는 '대안(조건을 만족하는 유사 생활권 단지 1~2개 제시)'으로 처리하세요.

[Self-Consistency(필수)]
- 웹 자동 수집/표준화는 1회만 수행하여 "고정 데이터셋"을 만드세요.
- 그 고정 데이터셋을 입력으로 Run A/B/C 3회 독립 분석을 수행하세요.
- 통합 규칙: 평균+범위를 제시하고, 2/3 동일 선택을 우선하며, 불일치 원인을 요약하고,
  범위가 큰 항목은 조건부/보류로 처리할 수 있도록 하세요.

[출력 형식]
0) 자동 수집 요약 + 출처 신뢰도
1) 비교 요약표(단지명 | 세대수 | 준공 | 상품성 | 입지 | 수급 | 가격·밸류 | 리스크 | 총점 | 신뢰도)
2) 단지별 상세 근거(각 항목: 근거 4~8줄 + 반례 가능성 + 검증 체크 2개)
3) 시나리오 3개(보수/중립/공격)
4) 최종 의사결정(매수/조건부/보류/대안 + 트리거 5개 + 검증 체크리스트 7개)

[누적 비교를 위한 로그 규칙(강제)]
- 매 분석 결과의 맨 끝에 반드시 "단지 카드(요약 로그)"를 출력하세요.
- 단지 카드는 아래 표준 필드로 고정하세요:
  (분석ID | 기준일 | 단지명 | 지역 | 세대수 | 준공 | 주력면적 | 최근12개월 매매거래수 | 총점 | 신뢰도 | 최종판정)
- 사용자가 "재비교"를 요청하면, 기존 단지 카드들을 우선 참조하세요.
  기준일이 오래되었거나 사용자가 원하면 웹 수집을 다시 수행하여 업데이트 비교하세요.

[시작]
지금부터 2단계 사용자 입력 폼을 출력하세요.
```

### AI 응답 확인하기

1단계 프롬프트를 보내면 AI는 바로 분석을 시작하지 않습니다.
대신 2단계에서 입력할 사용자 폼 양식을 출력합니다.
이것은 프롬프트에서 "입력 완료 전 분석 금지" 규칙을 지정했기 때문입니다.

### 확인할 포인트

| 확인 항목                     | 기준                                | 문제 시 대처                                    |
| ----------------------------- | ----------------------------------- | ----------------------------------------------- |
| AI가 폼 양식을 출력했는가     | 필수 11개 항목이 포함된 폼이 출력됨 | 1단계 프롬프트 전체가 복사되었는지 확인         |
| 바로 분석을 시작하지 않았는가 | 폼 출력 후 대기 상태                | "사용자 입력 폼을 먼저 출력해주세요"라고 재요청 |

---

## 2단계 입력 폼으로 나의 조건 제출하기

AI가 폼 양식을 출력했다면, 이제 내 조건을 채워 제출할 차례입니다.
모든 항목을 빠짐없이 입력해야 AI가 분석을 시작합니다.
누락된 항목이 있으면 AI가 추가 질문으로 보완을 요청합니다.

### 프롬프트를 보내기 전에

2단계 폼은 대괄호 안의 내용을 내 조건에 맞게 바꿔 입력합니다.
같은 폼 구조를 유지하면서 조건만 수정하면 되므로, 다른 단지를 비교할 때도 재사용할 수 있습니다.

> ❓ **궁금해요 Self-Consistency란?**
>
> **Self-Consistency** — AI에게 같은 데이터셋을 기반으로 분석을 3회 독립적으로 수행시키는 기법입니다.
> 3번 모두 같은 결론이 나오면 그 분석은 신뢰할 수 있고, 매번 다른 결론이 나오면 해당 항목에 불확실성이 있다는 의미입니다.
> 비교 요약표에서 점수 옆의 괄호(예: 4.0\~4.1)가 바로 이 3회 분석의 최소\~최대 범위입니다.

### 실행 방법

1. 1단계 프롬프트를 보낸 같은 대화창에서 계속 진행합니다. 새 창을 열면 처음부터 다시 해야 합니다.
2. 아래 2단계 입력 폼을 복사해서, 대괄호 안의 내용을 내 조건으로 바꿉니다.
3. 채운 폼을 그대로 붙여넣고 전송합니다.
4. AI가 필수 항목을 모두 확인하면 웹 수집 → Run A/B/C 분석을 순서대로 수행합니다.

### 프롬프트

```
[사용자 입력 폼 | 전 항목 필수]
분석 기준일: [YYYY-MM-DD]
투자 목적: [시세차익/월세수익/실거주]
```

## 두 개 이상의 AI로 결과를 교차 확인하는 방법

같은 프롬프트를 ChatGPT와 Gemini, 또는 Claude에 각각 입력해 결과를 비교하면 신뢰도가 높아집니다.
하나의 AI만 사용하면 해당 AI의 학습 데이터 편향이나 최신 데이터 부재가 그대로 결론에 반영될 수 있습니다.
두 개 이상의 AI를 교차 활용하면 서로 다른 관점에서 분석을 받아 보다 균형 잡힌 판단이 가능합니다.

아래는 같은 프롬프트를 ChatGPT와 Gemini에 각각 입력했을 때의 비교 예시입니다.

| 항목                  | ChatGPT              | Gemini                  |
| --------------------- | -------------------- | ----------------------- |
| **최종 선택**   | 민락푸르지오         | 민락푸르지오            |
| **결론 강도**   | 조건부 매수          | 강력 매수 권고          |
| **핵심 강조점** | 상승여력/밸류 분석   | 자금 안전성/월 상환액   |
| **자이 평가**   | 입지 최고, 밸류 낮음 | 매수 불가능 (예산 초과) |

두 AI 모두 같은 단지를 1순위로 추천했지만, 그 이유가 다릅니다.
ChatGPT는 가격 대비 성능(밸류)을 근거로 제시했고, Gemini는 자금 안전성을 근거로 제시했습니다.

결과 해석의 원칙은 다음과 같습니다.

- **결론이 같으면**: 신뢰도가 높습니다. 두 AI가 서로 다른 관점에서 같은 결론에 도달했으므로, 해당 단지가 유리할 가능성이 큽니다.
- **결론이 다르면**: 양쪽의 근거를 각각 검토해야 합니다. 어떤 AI가 어떤 데이터를 기반으로 판단했는지 확인하고, 직접 검증이 필요합니다.
- **근거가 다르면**: 설령 결론이 같더라도 양쪽 관점을 모두 고려해야 합니다. 한쪽은 밸류를 강조하고 다른 쪽은 안전성을 강조했다면, 두 관점 모두 유효한 판단 기준이 됩니다.

## AI 결과를 받은 후 직접 검증해야 할 항목

AI 분석은 빠르게 비교표와 결론을 정리해 주지만, 구체적인 수치나 사실 관계에서 틀릴 수 있습니다.
특히 실거래가, 역 거리, 입주물량은 AI 오류 가능성이 높은 항목입니다.
AI 결과는 1차 정리용으로 활용하고, 최종 확인은 반드시 공식 사이트에서 직접 합니다.

| 검증 항목 | AI 오류 가능성        | 검증 방법                            |
| --------- | --------------------- | ------------------------------------ |
| 실거래가  | 높음 (오래된 데이터)  | 국토부 실거래가 공개시스템, 호갱노노 |
| 역 거리   | 중간 (지도 기반 추정) | 네이버 지도 도보 측정                |
| 입주물량  | 높음 (일정 변동)      | 분양공고, 지자체 홈페이지            |
| 대출 한도 | 중간 (규제 변동)      | 은행 사전 심사                       |
| 학군      | 낮음                  | 학교알리미, 지역 맘카페              |

AI가 "역까지 도보 10분"이라고 했더라도 실제로 걸어보면 오르막이라 15분이 걸릴 수 있습니다.
AI가 제공한 실거래가가 학습 시점 이후의 최신 거래를 반영하지 못했을 수도 있습니다.

> ⚠️ **주의:** AI 분석 결과의 수치는 반드시 공식 출처와 교차검증해야 합니다. 특히 규제 관련 사항은 전문가 상담이 필요합니다.

## 최종 의사결정을 표로 정리하는 방법

AI 분석과 직접 검증을 마치면 결론을 아래 형식으로 정리합니다.

| 항목                | 결과                           |
| ------------------- | ------------------------------ |
| **1순위**     | [분석을 통해 선정된 단지명]    |
| **결정 유형** | 매수 / 조건부 매수 / 보류      |
| **트리거**    | [이 조건이 충족되면 매수 진행] |
| **다음 단계** | 실습 ㉓ 적정가 분석            |

의정부 민락동 3개 단지를 비교한 결과, 민락푸르지오가 1순위로 선정된 경우라면 다음과 같이 정리됩니다.

| 항목                | 결과                           |
| ------------------- | ------------------------------ |
| **1순위**     | 민락푸르지오                   |
| **결정 유형** | 조건부 매수                    |
| **트리거**    | 84타입 5.8억 이하 매물 확인 시 |
| **다음 단계** | 실습 ㉓ 적정가 분석            |

"어느 단지?"라는 질문에 대한 답을 얻었습니다.
다음 단계는 선정된 단지가 현재 가격에 매수할 만한 가치가 있는지, 즉 적정가인지 확인하는 것입니다.

## 핵심 정리

- **5가지 평가지표**: 상품성(20%)·입지(30%)·수급(20%)·가격밸류(15%)·리스크(15%)로 단지를 종합 채점합니다. 리스크 점수는 안전할수록 높습니다.
- **Self-Consistency**: AI가 동일 데이터셋으로 3회 독립 분석한 결과의 범위가 좁을수록 신뢰도가 높고, 범위가 넓은 항목은 직접 검증이 필요합니다.
- **교차 검증**: 두 개 이상의 AI에 같은 프롬프트를 입력해 결론과 근거를 비교하면 편향을 줄일 수 있습니다.
- **직접 검증 필수**: AI 결과는 1차 정리용입니다. 실거래가·역 거리·입주물량은 공식 사이트에서 반드시 확인합니다.
