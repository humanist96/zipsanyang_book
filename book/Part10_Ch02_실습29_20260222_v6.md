# 실습 ㉙: AI 기반 매도 전략 수립

Part 2~8에서 매수를 위한 분석 체계를 갖추었습니다.
이제 시선을 반대쪽으로 돌립니다. "잘 사는 것"만큼 중요한 것이 "제값에 파는 것"입니다.
이 실습에서는 2단계 프롬프트 구조를 활용하여, 내 점유 형태(본인 거주/세입자 거주)에 맞는 맞춤형 매도 전략을 AI에게 요청합니다.

**학습 목표**

- 2단계 프롬프트 구조(공용 프롬프트 + 상황별 입력 폼)로 맞춤형 매도 전략을 도출합니다
- 점유 형태(본인 거주/세입자 거주)에 따라 적합한 입력 폼을 선택하고 작성합니다
- AI 분석 결과에서 가격 전략, 타임라인, 리스크 관리 항목을 해석하고 실행에 옮깁니다

**핵심 키워드**
매도 전략 / 2단계 프롬프트 / 점유 형태 / 제시가-목표가-마지노선 / 매도 병목 / 타임라인 / 오감 마케팅

---

## 매수 분석과 매도 전략은 완전히 다른 작업이다

Part 7~8에서 배운 매수 분석은 "어떤 단지를, 얼마에, 언제 살 것인가"를 판단하는 과정이었습니다.
매도 전략은 방향이 반대입니다. "이 집을, 누구에게, 얼마에, 어떻게 팔 것인가"를 설계하는 과정입니다.

```
매수 분석: 시장 → 지역 → 단지 → 가격 → 매수 결정    (좁혀가는 과정)
매도 전략: 내 집 → 매수자 타겟 → 가격 설계 → 마케팅 → 계약  (넓혀가는 과정)
```

매수는 시장 전체에서 하나를 고르는 작업이지만, 매도는 내 집 하나를 시장에 내놓고 매수자를 끌어오는 작업입니다.
따라서 매도 전략에는 매수 분석에 없던 요소들이 추가됩니다.

| 매수 분석에서 다룬 요소  | 매도 전략에서 추가되는 요소  |
| ------------------------ | ---------------------------- |
| 적정가 범위, 시장 사이클 | 제시가, 양보 조건            |
| 단지 비교, 입지 분석     | 쇼잉 동선, 홈스테이징        |
| 과거 수익률, 거래량 추이 | 중개사 파트너십, 매수자 심리 |
| 임장, 현장 검증          | 세입자 협조, 인도 일정 설계  |

특히 세입자가 거주 중인 경우에는 매도 전략의 난이도가 크게 올라갑니다.
세입자의 퇴거 의사, **갱신요구권** `<sup>`세입자가 계약 만료 시 한 번에 한해 2년 연장을 요구할 수 있는 권리(주택임대차보호법 제6조의3)`</sup>` 행사 여부, 보증금 반환 계획까지 고려해야 하기 때문입니다.
이 실습의 프롬프트는 본인 거주와 세입자 거주를 구분하여 각각에 맞는 전략을 도출하도록 설계되었습니다.

---

## 내 상황에 맞는 입력 폼 선택하기

이 실습의 프롬프트는 2단계로 구성됩니다.
1단계는 AI에게 역할과 분석 규칙을 부여하는 공용 프롬프트이고, 2단계는 내 상황을 입력하는 폼입니다.
2단계 폼은 점유 형태에 따라 두 가지로 나뉩니다.

| 점유 형태                      | 사용할 폼 | 핵심 차이                                |
| ------------------------------ | --------- | ---------------------------------------- |
| 내가 살고 있다 (실거주)        | STEP 2-A  | 내부 상태, 인도 시기, 수리 예산 중심     |
| 세입자가 살고 있다 (전세/월세) | STEP 2-B  | 임대차 핵심 6개 항목, 세입자 협조도 중심 |
| 공실 (비어 있음)               | STEP 2-A  | 본인 거주 폼으로 충분                    |

STEP 2-A의 필수 항목은 10개, STEP 2-B는 14개입니다.
세입자 거주 매도가 더 많은 정보를 요구하는 이유는, 임대차 관계에서 발생하는 변수(갱신요구권, 보증금 반환, 쇼잉 협조 등)를 AI가 반영해야 하기 때문입니다.

## 1단계 공용 프롬프트로 분석 환경 설정하기

1단계 프롬프트는 AI에게 매도 전략가 역할을 부여하고, 분석 규칙과 출력 형식을 설정합니다.
한 번만 입력하면 같은 대화 세션에서 2단계 폼을 바꿔가며 반복 분석할 수 있습니다.

### 프롬프트를 보내기 전에

이 프롬프트에는 3가지 핵심 규칙이 포함되어 있습니다.

첫째, **입력 검증 게이트**입니다. 2단계 폼의 필수 항목이 누락되면 AI가 분석을 시작하지 않고 누락 항목만 안내합니다.
Part 7 실습에서 사용한 "입력 완료 전 분석 금지" 규칙과 같은 원리입니다.

둘째, **면적 표준화 규칙**입니다. "34평", "84타입", "84.98㎡"처럼 다양한 표현을 AI가 전용면적(㎡) 기준으로 표준화합니다.
다만 매핑 근거가 없으면 확정 대신 후보/범위로 제시하며, 추가 확인 항목을 함께 안내합니다.

셋째, **할루시네이션 방지 규칙**입니다. 사용자가 실거래·매물 데이터를 제공하지 않았으면 AI가 가격 숫자를 확정처럼 제시하지 않고, 가격 설계 방법(제시가/목표가/마지노선/양보조건) 중심으로 전략을 제시합니다.

### 실행 방법

1. ChatGPT에서 새 대화창(New Chat)을 엽니다.
2. 최신모델을 선택하고, 웹 검색 기능을 켭니다.
3. 아래 1단계 프롬프트 전체를 복사해서 붙여넣습니다.
4. AI가 확인 메시지를 출력하면 2단계로 넘어갑니다.

💡 **Tip.** 이 프롬프트는 ChatGPT 외에 Claude, Gemini에서도 사용할 수 있습니다. 어떤 AI 모델이든 최신모델과 웹 검색 기능을 켜고 실행하는 것이 효과적입니다.

### 프롬프트

```
# 📘 [STEP 1 | 공용 프롬프트]

[ROLE]
당신은 상위 1%의 "한국 아파트 매도 전략가 + 리스크 매니저 + 협상/실무 코치"입니다.
사용자의 점유 형태(실거주/세입자), 정책·시장 환경, 매수자 심리를 종합해
"가장 안전하면서 제값에 파는 실행 전략(가격·협상·마케팅·중개·타임라인·특약·플랜B)"을 제시합니다.

[2-STEP WORKFLOW | 절대 규칙]
- STEP 1: 이 공용 프롬프트는 1회만 입력됩니다.
- STEP 2: 사용자는 STEP 2-A(실거주) 또는 STEP 2-B(세입자) 중 하나를 제출합니다.
- 사용자의 메시지에 STEP 2-A/B 내용이 이미 포함되어 있으면, 폼을 다시 출력하지 말고 "입력 검증"부터 시작하세요.

[입력 검증 게이트 | 통과 전 분석 금지]
- STEP 2의 필수 항목이 누락되면:
  1) 누락 항목 표만 출력
  2) 분석/전략/가격 밴드 생성 금지
  3) "누락만 채워 재제출" 안내
- 단, 필수 항목이 충족되면 즉시 분석을 시작합니다.

[CRITICAL RULES: 할루시네이션 방지 및 리스크 관리]
1) 면적 표준화(최우선 강제)
- 기준은 반드시 "전용면적(㎡)"입니다.
- "평(예: 34평)"만 있을 때 1평=3.3㎡로 전용을 단정하는 행위 금지.
- 공식 매핑 근거(분양공고/평면도/공급면적표/공식 단지정보 문구 등)가 없으면:
  - "전용 후보 2~3개 또는 전용 범위"로 제시하고,
  - 신뢰도(높음/중간/낮음) + 필요한 추가 증빙을 함께 제시하세요.
- 반드시 아래 4가지를 출력:
  (a) 원문 면적 입력 (b) 전용 표준화 결과(확정/후보/범위)
  (c) 신뢰도+이유 (d) 추가로 필요한 근거

2) 세금/정책 단정 금지
- 세금/규제는 '현재 정책 기준의 시나리오'로만 제시하고,
  세무사/은행/공식 확인 필요를 명시하세요.

3) 데이터 없을 때 "가격 숫자" 만들지 않기
- 사용자가 실거래/현재매물 데이터를 제공하지 않았으면,
  "가격 밴드 숫자"는 확정처럼 말하지 말고:
  - (가능하면) 가정 기반의 넓은 범위로만 제시하거나,
  - 숫자 대신 "가격 설계 방법(제시가/목표가/마지노선/양보조건)" 중심으로 제시하세요.

4) 점유 형태별 병목 진단(필수)
- 실거주: 정리/홈스테이징/쇼잉 동선/인도 일정
- 세입자: 만료/갱신요구권/보증금 반환 플랜/승계 vs 공실/협조 설계

[마케팅 및 협상 코칭 규칙]
- (가격 구조) "제시가 / 목표가 / 마지노선 / 양보 조건(대신 받을 조건)" 4단 구조로 제시하세요.
- (오감 마케팅) 조명·향·온도·비우기·소음·환기를 상황에 맞게, 과하지 않게 제안하세요.
- (중개사 파트너십) 불법 리베이트 금지. 법정 보수 범위 내에서
  "피드백 루프(주간 리포트), 콘텐츠 강화, 오픈하우스 운영, 동일 수요권 네트워크 확장"으로 우선 브리핑을 유도하세요.

[OUTPUT FORMAT | 반드시 아래 순서]
## 1. 📊 입력 요약 및 팩트 체크(표)
- 상황 요약(3줄)
- 면적 표준화 결과(확정/후보/범위 + 신뢰도 + 필요한 근거)
- 매도 병목(리스크) 1순위

## 2. 💡 정책/시장 반영 3대 매도 시나리오(표)
| 시나리오 | 타겟 매수자 | 추천 가격 구조(제시가/목표가/마지노선) | 네고 방어 논리 | 실패 시 플랜B |
|---|---|---|---|---|
| S1 빠른 매도(기간 우선) | | | | |
| S2 최고가(가격 우선) | | | | |
| S3 균형(리스크 최소화) | | | | |

## 3. 🎯 실전 마케팅 & 중개 전략(리스트)
- 쇼잉/공간 연출 3가지
- 중개사 파트너십 2가지(합법 범위)

## 4. 🗓️ 실행 타임라인 & 소통 스크립트(표+짧은 스크립트)
- 준비 → 노출/쇼잉 → 가계약/본계약 → 잔금/인도
- 스크립트: 중개사 / 매수자 / (해당 시) 세입자

## 5. 🛡️ 계약 리스크 관리 & 특약 아이디어(리스트 2~4개)
- 대출 불발/잔금 지연/하자/인도·명도/보증금 관련 방어

## 6. 🚨 레드팀(최악 대비) + 정책 스트레스 테스트(리스트)
- 규제 강화/입주물량/거래절벽 발생 시 즉시 플랜B

## 7. 추가 질문(있다면 최대 5개)
```

### AI 응답 확인하기

1단계 프롬프트를 보내면 AI는 바로 분석을 시작하지 않습니다.
대신 STEP 2 입력 폼의 선택 안내(실거주는 STEP 2-A, 세입자 거주는 STEP 2-B)를 출력합니다.
이것은 프롬프트에서 "입력 검증 게이트" 규칙을 지정했기 때문입니다.

| 확인 항목                           | 기준                             | 문제 시 대처                                         |
| ----------------------------------- | -------------------------------- | ---------------------------------------------------- |
| AI가 STEP 2 안내를 출력했는가       | 실거주/세입자 선택 안내가 포함됨 | 1단계 프롬프트 전체가 복사되었는지 확인              |
| 바로 분석을 시작하지 않았는가       | 안내 출력 후 대기 상태           | "STEP 2 입력 폼 안내를 먼저 출력해주세요"라고 재요청 |
| OUTPUT FORMAT 7개 섹션을 인지했는가 | 확인 메시지에 출력 형식 언급     | 없어도 무방, 2단계 결과에서 확인 가능                |

## 2단계 입력 폼으로 매도 조건 제출하기

1단계에서 AI가 안내를 출력했다면, 이제 내 상황에 맞는 2단계 폼을 채워 제출합니다.
본인이 거주 중이면 STEP 2-A를, 세입자가 거주 중이면 STEP 2-B를 선택합니다.

### 프롬프트를 보내기 전에

2단계 폼은 대괄호 안의 내용을 내 조건에 맞게 바꿔 입력합니다.
같은 폼 구조를 유지하면서 조건만 수정하면 되므로, 결과가 마음에 들지 않을 때 2단계만 다시 제출할 수 있습니다.
1단계는 다시 입력할 필요가 없습니다.

면적은 평소 사용하는 표현 그대로 입력해도 됩니다.
"34평", "84타입", "84A", "84.98㎡" 모두 가능합니다.
AI가 전용면적(㎡) 기준으로 표준화하되, 확정이 어려우면 후보/범위와 함께 추가 확인 항목을 안내합니다.

### 실행 방법

1. 1단계 프롬프트를 보낸 같은 대화창에서 계속 진행합니다. 새 창을 열면 처음부터 다시 해야 합니다.
2. 아래 STEP 2-A 또는 STEP 2-B 중 내 상황에 맞는 폼을 선택합니다.
3. 대괄호 안의 내용을 내 조건으로 바꿔 붙여넣고 전송합니다.
4. 필수 항목이 모두 충족되면 AI가 7개 섹션 형식으로 매도 전략을 출력합니다.

### 프롬프트 — STEP 2-A: 본인 거주 매도용

```
# 📘 [STEP 2-A | 본인 거주 매도용 입력 폼]
(현재 집에 본인이 거주 중일 때만)

[필수]
- 분석 기준일:
- 지역 및 단지명(시/구/동 포함):
- 면적/타입(평소 부르는 대로):
- (알면) 전용면적(㎡):
- 희망 매도 완료 시점(계약 또는 잔금):
```

### AI 응답 확인하기

폼을 제출하면 AI가 필수 항목을 검증한 뒤 7개 섹션으로 매도 전략을 출력합니다.
STEP 2-A(본인 거주)로 입력한 경우, 다음과 같은 형태의 결과가 나옵니다.

**입력 예시 (1주택 실거주, 서울 마포구)**


## AI가 제시해도 반드시 검증이 필요한 3가지

AI가 7개 섹션으로 전략을 제시하더라도, 아래 3가지 영역은 반드시 전문가 또는 공식 기준으로 최종 확인해야 합니다.

| 검증 영역                                  | AI 한계                                                 | 검증 방법                                  |
| ------------------------------------------ | ------------------------------------------------------- | ------------------------------------------ |
| **세금(양도세/비과세 요건)**         | 보유 기간, 거주 기간, 다주택 중과 등 개인별 변수가 많음 | 세무사 상담 또는 국세청 홈택스             |
| **규제 적용(규제지역/토지거래허가)** | 시점·지역·예외에 따라 결과가 달라짐                   | 해당 구청 또는 국토부 규제지역 조회        |
| **임대차 분쟁 가능성(세입자 거주)**  | 실제 계약서 문구, 대화 맥락에 따라 리스크가 달라짐      | 법률구조공단 또는 주택임대차분쟁조정위원회 |

Part 7에서 실거래가와 호가를 교차검증했던 것처럼, 매도 전략에서도 AI 결과의 숫자와 판단은 1차 참고자료로 활용하고 최종 확인은 직접 수행하는 원칙을 유지합니다.

> ⚠️ **주의:** AI가 "비과세 가능"이라고 판정했더라도, 보유 기간·거주 기간·주택 수 산정에서 예외가 적용될 수 있습니다. 양도소득세는 반드시 세무사에게 확인한 뒤 매도를 진행합니다.

## 결과가 마음에 들지 않을 때 재입력하는 방법

AI 분석 결과가 기대와 다르거나 조건이 바뀌었을 때는 2단계 폼만 수정하여 다시 제출하면 됩니다.
1단계 공용 프롬프트는 이미 적용되어 있으므로 다시 입력할 필요가 없습니다.

재입력이 필요한 대표적인 상황은 3가지입니다.

| 상황                        | 대응                                                |
| --------------------------- | --------------------------------------------------- |
| 마지노선을 변경하고 싶다    | 2단계 폼에서 마지노선만 수정하여 재제출             |
| 매도 완료 시점이 앞당겨졌다 | 희망 매도 완료 시점을 수정하여 재제출               |
| 세입자가 퇴거 의사를 밝혔다 | STEP 2-B에서 퇴거 의사를 "있음"으로 수정하여 재제출 |

💡 **Tip.** 같은 조건으로 ChatGPT와 Claude에 각각 입력하면 서로 다른 관점의 전략을 비교할 수 있습니다. Part 7 실습 ㉒에서 두 AI의 결과를 교차 확인했던 것과 같은 원리입니다.

## 오늘 당장 실행할 3가지

AI 분석을 마치면 다음 3가지를 바로 실행합니다.

첫째, 중개사에게 **피드백 루프**를 요청합니다.
문의 수, 네고 요청, 매물 조회수, 매수자 반응을 일정 주기(주 1회 권장)로 보고받는 구조를 만듭니다.
이 루프가 없으면 "왜 안 팔리는지" 원인을 파악할 수 없습니다.

둘째, 내 **마지노선과 양보 조건을 문장으로 고정**합니다.
"22억 이하 불가, 단 인도일 2주 연장은 가능"처럼 구체적으로 적어둡니다.
매수자의 네고 압박에 감정이 흔들리는 것을 방지하는 장치입니다.

셋째, 세입자 거주라면 **인도·보증금·승계 중 1순위를 먼저 결정**합니다.
세입자와의 협의에서 가장 중요한 카드가 무엇인지 정해야 구체적인 대화가 가능합니다.

## 핵심 정리

- **2단계 프롬프트 구조**: 공용 프롬프트로 분석 규칙을 설정하고, 점유 형태에 맞는 폼(STEP 2-A/B)을 제출하면 맞춤형 매도 전략이 도출됩니다.
- **제시가-목표가-마지노선-양보조건**: 가격 전략은 4단 구조로 설계하여 협상 과정에서 단계별 방어선을 유지합니다.
- **3가지 필수 검증**: 세금, 규제, 임대차 분쟁은 AI 결과와 무관하게 반드시 전문가 또는 공식 기준으로 확인합니다.
- **피드백 루프**: 중개사에게 주간 리포트를 요청하여 매도 전략의 효과를 지속적으로 점검합니다.
