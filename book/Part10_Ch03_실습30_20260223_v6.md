# 실습 ㉚: 계약 리스크 방어와 맞춤형 특약 작성

실습 ㉙에서 매도 전략을 수립했다면, 이제 계약 테이블에서 발생할 수 있는 돌발 변수에 대비합니다.
매수자의 대출 불발, 잔금 지연, 하자 분쟁 등 초보 매도자가 미처 생각하지 못한 리스크를 AI로 진단하고, 거래 단계별 방어 특약 아이디어와 체크리스트를 도출합니다.

**학습 목표**

- 2단계 프롬프트 구조로 계약 리스크를 진단하고 방어 특약 아이디어를 도출합니다
- 거래 단계(가계약~잔금)별 체크리스트를 활용하여 리스크를 사전에 차단합니다
- AI가 제안한 특약을 공인중개사와 협의하여 실전 계약에 반영하는 방법을 이해합니다

**핵심 키워드**
계약 리스크 / 방어 특약 / 거래 단계별 체크리스트 / 대출 불발 / 인도·명도 / 커뮤니케이션 스크립트

---

## 표준 계약서만으로는 특수 상황을 방어할 수 없다

매수자가 나타나 가계약금을 입금하겠다는 연락을 받으면, 그동안의 긴장감이 한순간에 풀립니다.
하지만 부동산 거래에서 진짜 리스크는 계약 단계에서 시작됩니다.

초보 매도자가 가장 많이 하는 실수는 계약서 작성을 공인중개사에게 전적으로 맡기는 것입니다.
통상적으로 사용하는 표준 계약서는 세입자 명도 지연, 매수자의 대출 불발, 갑작스러운 하자 보수 요구 같은 특수 상황을 모두 방어하지 못합니다.
계약서에 도장이 찍히는 순간, 모든 법적·금전적 책임은 매도자의 몫이 됩니다.

> '잔금 시 하자는 협의한다'

이런 두리뭉실한 문구는 분쟁 시 오히려 독이 됩니다.
날짜, 금액, 책임 소재가 명확한 방어 특약이 필요합니다.

이 실습의 프롬프트는 나의 거래 상황을 입력하면 AI가 핵심 리스크 TOP5를 진단하고, 각 리스크에 대응하는 특약 아이디어(의도·핵심 조건)와 거래 단계별 체크리스트를 출력합니다.
또한 중개사·매수자·세입자에게 특약을 제안할 때 사용할 수 있는 커뮤니케이션 스크립트도 함께 제공합니다.

> ⚠️ **주의:** AI가 제안하는 특약은 법적 효력이 보장된 최종 문구가 아닙니다. '의도와 핵심 조건'을 참고하여 반드시 공인중개사 또는 법률 전문가의 검토를 받은 뒤 계약서에 반영합니다.

---

## 1단계 공용 프롬프트로 계약 방어 환경 설정하기

실습 ㉙과 동일한 2단계 프롬프트 구조를 사용합니다.
1단계에서 AI에게 계약 리스크 매니저 역할을 부여하고, 2단계에서 실제 거래 상황을 입력합니다.

### 프롬프트를 보내기 전에

이 프롬프트의 핵심 규칙은 2가지입니다.

첫째, **법적 효력 단정 금지**입니다. AI는 특약의 '의도와 핵심 조건'을 제안하지만, 법적 효력을 확정하지 않습니다.
모든 출력에 "공인중개사/법률 전문가 검토 필요"가 명시됩니다.

둘째, **입력 검증 게이트**입니다. 실습 ㉙과 동일하게, 필수 항목이 누락되면 AI가 분석을 시작하지 않고 누락 항목만 안내합니다.

### 실행 방법

1. ChatGPT에서 새 대화창(New Chat)을 엽니다.
2. 최신모델을 선택하고, 웹 검색 기능을 켭니다.
3. 아래 1단계 프롬프트 전체를 복사해서 붙여넣습니다.
4. AI가 확인 메시지를 출력하면 2단계로 넘어갑니다.

💡 **Tip.** 이 프롬프트는 실습 ㉙과 별도의 대화창에서 실행합니다. 실습 ㉙ 대화에 이어서 입력하면 기존 매도 전략가 역할과 충돌할 수 있습니다.

### 프롬프트

```
# [계약 리스크 방어: 특약 아이디어 & 체크리스트 | STEP 1]

[ROLE]
당신은 "아파트 매도 계약 리스크 매니저(매도자 편) + 실무 코치"입니다.
목표는 분쟁·지연·대출불발·인도/명도·세입자 변수로 인한 손실을 줄이는 것입니다.

[2-STEP WORKFLOW]
- STEP 1(지금): 규칙/검증/출력 형식을 설정합니다.
- STEP 2: 사용자가 계약 상황을 입력하면, 검증 통과 시에만 '특약 아이디어/체크리스트/질문' 중심으로 출력합니다.

[절대 규칙]
- 법적 효력을 단정하지 마세요.
- 최종 문구는 반드시 "공인중개사/법률 전문가 검토 필요"를 명시하세요.
- 특약은 '문구 확정'이 아니라 '의도·핵심 조건 제안' 중심으로 작성하세요.

[입력 검증 게이트]
- 필수 항목 누락 시:
  → 누락 항목 표만 출력하고 분석 시작 금지.

[OUTPUT FORMAT | 순서 고정]
1) 입력 요약 & 핵심 리스크 TOP5
2) 리스크별 특약 의도(핵심 조건) + 필요한 증빙
3) 단계별 체크리스트(가계약→본계약→중도금→잔금)
4) 커뮤니케이션 스크립트(중개사/매수자/세입자 해당 시)
5) 추가 확인 질문(최대 7개)
```

### AI 응답 확인하기

1단계 프롬프트를 보내면 AI는 STEP 2 입력 안내를 출력합니다.
실습 ㉙과 마찬가지로 입력 검증 게이트가 작동하므로, 2단계 폼을 제출하기 전까지 분석을 시작하지 않습니다.

### 확인할 포인트

| 확인 항목                               | 기준                            | 문제 시 대처                            |
| --------------------------------------- | ------------------------------- | --------------------------------------- |
| AI가 STEP 2 안내를 출력했는가           | 본인 거주/세입자 거주 선택 안내 | 1단계 프롬프트 전체가 복사되었는지 확인 |
| "법적 효력 단정 금지" 규칙을 인지했는가 | 확인 메시지에 관련 언급         | 없어도 무방, 2단계 결과에서 확인 가능   |

---

## 2단계 입력 폼으로 계약 상황 제출하기

1단계 안내를 확인했다면, 내 거래 상황에 맞는 2단계 폼을 채워 제출합니다.
본인 거주 중이면 STEP 2-A를, 세입자가 거주 중이면 STEP 2-B를 선택합니다.

### 프롬프트를 보내기 전에

2단계 폼은 거래 단계, 걱정되는 리스크, 가격/조건, 하자/옵션 정보를 입력합니다.
대괄호 안의 입력 가이드는 삭제하지 않아도 됩니다. AI가 가이드와 실제 입력을 구분합니다.

특히 **"가장 걱정되는 리스크 TOP3"** 항목이 중요합니다.
여기에 입력한 리스크를 중심으로 AI가 특약 아이디어를 집중 설계합니다.

### 실행 방법

1. 1단계 프롬프트를 보낸 같은 대화창에서 계속 진행합니다.
2. 아래 STEP 2-A 또는 STEP 2-B 중 내 상황에 맞는 폼을 선택합니다.
3. 대괄호 안의 내용을 내 조건으로 바꿔 붙여넣고 전송합니다.
4. 필수 항목이 모두 충족되면 AI가 5개 섹션으로 계약 리스크 분석을 출력합니다.

### 프롬프트 — STEP 2-A: 본인 거주용

```
# [계약 리스크 방어 | STEP2-A 본인 거주용]

※ [] 안의 문장은 입력 가이드입니다. 삭제하지 않아도 됩니다.

[필수]

- 분석 기준일(YYYY-MM-DD):
  [입력 가이드: 오늘 날짜
   예: 2026-03-15]

- 거래 단계:
  [입력 가이드: 아래 중 하나 선택
   예: 가계약 직전
   선택지: 매물 준비 / 문의·네고 중 / 가계약 직전 / 가계약 후 / 본계약 직전 / 본계약 후 / 중도금 후 / 잔금 직전]

- 희망 잔금일(또는 기간):
  [입력 가이드: 날짜(YYYY-MM-DD) 또는 "계약 후 n개월"
   예: 2026-05-30
   예: 계약 후 2개월 이내]

- 인도 가능 시점(잔금일 즉시 / 유예 필요(기간)):
  [입력 가이드: 인도(집 비워주기) 가능 조건
   예: 잔금일 즉시 가능
   예: 잔금 후 2주 유예 필요]

- 가장 걱정되는 리스크 TOP3:
  [입력 가이드: 이번 거래에서 특히 불안한 1~3개
   예: ① 매수자 대출 불발 ② 잔금 지연 ③ 하자 트집/분쟁
   예: ① 계약 파기 ② 옵션 포함범위 분쟁 ③ 잔금일 변경 요구]

[가격/조건]

- 현재 합의 가격(또는 네고 상황):
  [입력 가이드: "현재 호가/합의 가격/추가 요구"를 짧게
   예: 10.3억 구두 합의, 2천만 원 추가 인하 요구 중
   예: 9.8억 제시 받았고 인도 유예 조건으로 밀고 있음]

- 매수자 대출 상황(사전승인/의존도 높음/모름):
  [입력 가이드: 알고 있는 수준만
   예: 사전승인 받았다고 함(서류 확인 전)
   예: 대출 의존도 높음(LTV 최대치 활용 예정)
   예: 모름]

- 잔금까지 기간(대략 개월 수):
  [입력 가이드: 기간을 개월로
   예: 약 2개월 / 약 3개월]

[하자/옵션]

- 최근 수리·교체 이력:
  [입력 가이드: 큰 항목 위주로(연도+내용)
   예: 2024 보일러 교체, 2025 도배·장판 교체
   예: 수리 없음 / 모름]

- 옵션 포함/제외 항목:
  [입력 가이드: 분쟁 잦은 것만이라도 구체적으로
   예: 시스템에어컨 포함, 붙박이장 포함, 커튼 제외
   예: 빌트인 냉장고 포함, 식기세척기 제외]

- 분쟁 가능성 있는 요소:
  [입력 가이드: '매수자가 트집 잡을 만한 포인트'가 있으면 적기
   예: 베란다 확장 관련 서류 확인 필요(불명)
   예: 과거 누수 이력 있음(현재는 수리 완료)
   예: 소음 민원 가능(도로변)]

[선택]

- 중개사 제안 특약(복붙 가능):
  [입력 가이드: 문구 그대로 붙여넣기
   예: "매수인의 대출 불가 시 계약 해제 가능" 특약 제안 받음]

- 우선순위(가격/속도/분쟁 최소/스트레스 최소):
  [입력 가이드: 중요도 순서로
   예: 분쟁 최소 > 가격 > 속도
   예: 가격 > 속도 > 스트레스 최소]

- 절대 피하고 싶은 상황:
  [입력 가이드: '이건 무조건 막아야 한다' 1~2개
   예: 잔금 직전 계약 파기
   예: 잔금 지연으로 이사 일정 붕괴]
```

### 프롬프트 — STEP 2-B: 세입자 거주용

```
# [계약 리스크 방어 | STEP2-B 세입자 거주용]

※ [] 안의 문장은 입력 가이드입니다. 삭제하지 않아도 됩니다.

[필수]

- 분석 기준일(YYYY-MM-DD):
  [입력 가이드: 오늘 날짜
   예: 2026-03-15]

- 거래 단계:
  [입력 가이드: 아래 중 하나 선택
   예: 본계약 직전
   선택지: 매물 준비 / 문의·네고 중 / 가계약 직전 / 가계약 후 / 본계약 직전 / 본계약 후 / 중도금 후 / 잔금 직전]

- 희망 잔금일(또는 기간):
  [입력 가이드: 날짜 또는 "계약 후 n개월"
   예: 2026-06-10
   예: 계약 후 3개월]

- 가장 걱정되는 리스크 TOP3:
  [입력 가이드: 세입자/보증금/인도/대출 관련 불안 1~3개
   예: ① 세입자 퇴거 번복 ② 보증금 반환 타이밍 ③ 잔금 지연
   예: ① 승계 거부 매수자 많음 ② 쇼잉 비협조 ③ 분쟁 위험]

[임대차 핵심]

- 임대차 형태(전세/월세/반전세):
  [입력 가이드: 하나 선택
   예: 전세]

- 보증금/월세:
  [입력 가이드: 숫자로 명확히
   예: 보증금 3억 / 월세 0
   예: 보증금 2억 + 월 80만 원]

- 계약 시작일 / 만료일:
  [입력 가이드: YYYY-MM-DD
   예: 2024-10-01 / 2026-10-01]

- 갱신요구권 이슈(있음/없음/모름):
  [입력 가이드: 알고 있는 수준만
   예: 아직 사용 안 함
   예: 사용함(2024년 1회)
   예: 모름]

- 세입자 퇴거 의사(있음/없음/모름):
  [입력 가이드: 현재 태도
   예: 만기 시 퇴거 예정(구두)
   예: 계속 거주 희망
   예: 불확실]

- 승계 가능 여부(가능/불가/협의):
  [입력 가이드: 승계 매도 허용 범위
   예: 승계 가능
   예: 승계 불가(공실 인도 조건만)
   예: 협의(가격/조건 따라)]

- 쇼잉 협조 난이도(쉬움/보통/어려움):
  [입력 가이드: 집 보여주기 협조 수준
   예: 보통(주말만 가능)
   예: 어려움(방문 자체를 꺼림)]

[가격/조건]

- 현재 합의 가격(있다면):
  [입력 가이드: 합의/제안/네고 상태
   예: 10.2억 제안 받음, 10.4억 희망
   예: 아직 가격 협의 전]

- 매수자 대출 상황(사전승인/의존도 높음/모름):
```

## 핵심 정리

- **방어 특약**: AI가 제안하는 특약은 '의도와 핵심 조건' 중심이며, 최종 문구는 반드시 공인중개사 또는 법률 전문가의 검토를 거칩니다.
- **거래 단계별 체크리스트**: 가계약부터 잔금까지 각 단계에서 확인할 사항을 사전에 정리하여 리스크를 차단합니다.
- **커뮤니케이션 스크립트**: 중개사·매수자·세입자에게 특약을 제안할 때 감정 충돌 없이 논리적으로 요청하는 대본을 활용합니다.
- **실습 ㉙과의 연결**: 매도 전략(실습 ㉙) → 계약 방어(이 실습) 순서로 진행하면 전략 수립부터 계약 마무리까지 일관된 흐름을 유지합니다.
