# 실습 ㉓: 시장 사이클을 고려한 적정가 분석

실습 ㉒에서 5가지 평가지표와 Self-Consistency로 후보 단지 1순위를 선정했습니다.
이제 남은 질문은 하나입니다. "이 단지를 지금 이 가격에 사도 되는가?"
이 실습에서는 2단계 프롬프트를 활용하여 현재 가격이 적정 범위에 있는지, 지금이 매수 타이밍인지 AI로 판단합니다.
1단계에서 분석 규칙과 출력 형식을 설정하고, 2단계에서 내 조건을 입력하면 AI가 DATA PACK 6개 표를 작성한 뒤 적정가를 산출합니다.

**학습 목표**

- 시장 사이클 위치(상승/고점/하락/저점/회복)를 기준으로 현재 가격의 의미를 해석합니다
- 실거래 밴드, 전세가율 역산, 비교단지 ㎡당 적용 3가지 모델로 적정가 범위를 산출합니다
- 갭투자 최적화 모드와 타이밍 매수 모드의 차이를 이해하고 본인 목적에 맞게 선택합니다
- AI 분석 결과에서 매수·보류 트리거를 뽑아 구체적인 행동 기준을 세웁니다

**핵심 키워드**
시장 사이클 / 적정가 범위 / 실거래 밴드 / 전세가율 역산 / 갭투자 최적화 / 타이밍 매수 / DATA PACK / 실행 트리거

## "이 가격이 비싼 건지 적정한 건지" 판단하기 어려운 이유

1순위 단지를 선정하고 나면 다음 질문은 자연스럽게 나옵니다.

'민락푸르지오 84㎡가 5.8억에 나왔는데, 이게 맞는 가격인가?'

단순히 최근 실거래가와 비교하는 것으로는 충분하지 않습니다.
같은 가격이라도 시장 사이클에 따라 의미가 완전히 달라지기 때문입니다.

```
상승장 ──▶ 고점 ──▶ 하락장 ──▶ 저점 ──▶ 회복장 ──▶ 상승장
              │                    │
              ▼                    ▼
        같은 5.8억이          같은 5.8억이
        비싼 가격              저렴한 가격
```

상승장 끝자락의 5.8억과 하락장 저점의 5.8억은 같은 숫자이지만 의미가 다릅니다.
사이클 위치에 따라 적정가 판단이 달라지기 때문에, "지금 얼마인가"가 아니라 "지금 시장이 어느 국면인가"를 함께 봐야 합니다.

적정가를 판단할 때는 5가지 요소를 종합적으로 살펴봐야 합니다.

| 분석 요소               | 내용                     | 왜 중요한가                               |
| ----------------------- | ------------------------ | ----------------------------------------- |
| **실거래 데이터** | 최근 12~24개월 거래 내역 | 실제로 얼마에 팔렸는지가 진짜 가격입니다  |
| **전세 정보**     | 전세가, 전세가율         | 갭투자 효율성 판단 기준입니다             |
| **비교단지**      | 주변 비슷한 단지 가격    | 상대적으로 비싼지 싼지 비교할 수 있습니다 |
| **거래량**        | 월별 거래 건수           | 시장 심리를 파악합니다 (거래절벽 vs 회복) |
| **정책/이벤트**   | 금리, 규제, 대규모 입주  | 가격에 영향을 줄 변수들입니다             |

> ❓ **궁금해요 전세가율이란?**
>
> 전세가율은 매매가 대비 전세가의 비율입니다.
> 매매가 10억 원짜리 아파트의 전세가가 6억 원이면 전세가율은 60%입니다.
> 전세가율이 높을수록 적은 자기자본으로 투자할 수 있어 레버리지 효과가 커집니다.
> 다만 70~80% 이상이면 집값 하락 시 전세금을 돌려주지 못하는 **깡통전세** — 전세금이 집값보다 높아지는 상황 — 리스크가 생깁니다.
> 60% 내외가 일반적으로 적정 수준으로 봅니다.

AI는 한 가지 방법이 아니라 3가지 모델로 적정가를 교차 산출합니다.

| 모델             | 산출 방식                       | 핵심 질문                            |
| ---------------- | ------------------------------- | ------------------------------------ |
| **모델 A** | 실거래 밴드 기반 (25%~75% 구간) | 최근 거래가가 분포하는 범위는?       |
| **모델 B** | 전세가율 역산 (55~65% 가정)     | 전세가율이 적정하려면 매매가는 얼마? |
| **모델 C** | 비교단지 ㎡당 가격 적용         | 주변 비슷한 단지 대비 적정 가격은?   |

3가지 모델의 중앙값이 최종 적정가 범위가 됩니다.
현재 판단 대상 가격이 이 범위 안에 있으면 "적정", 범위 아래면 "저평가", 범위 위면 "고평가"로 판단합니다.

## 분석 모드 선택하기: 갭투자 vs 타이밍 매수

이 실습의 프롬프트는 두 가지 분석 모드를 지원합니다.
본인의 투자 목적에 맞는 모드를 먼저 결정해야 합니다.

| 분석 모드               | 핵심 지표      | 적합한 사람                     |
| ----------------------- | -------------- | ------------------------------- |
| **갭투자 최적화** | 전세가율 중심  | 전세를 끼고 매수, 시세차익 목적 |
| **타이밍 매수**   | 가격 위치 중심 | 현금 매수 또는 실거주 목적      |

갭투자 모드에서는 전세가율이 리스크 성향에 맞는 임계값을 충족하는지가 핵심입니다.

| 리스크 성향 | 전세가율 임계값 | 의미                          |
| ----------- | --------------- | ----------------------------- |
| 보수        | 65% 이상        | 안전하게 투자, 자금 부담 적음 |
| 중립        | 60% 이상        | 적정 수준                     |
| 공격        | 55% 이상        | 레버리지 낮지만 감수          |

타이밍 매수 모드에서는 적정가 대비 현재 가격 위치, 실거래 추세, 거래량을 종합적으로 봅니다.

모드를 결정했다면 이제 2단계 프롬프트를 실행합니다.

## 1단계 프롬프트로 분석 규칙 설정하기

이 프롬프트는 2단계로 구성됩니다.
1단계는 AI에게 분석 운영 규칙, 출력 형식, 검색 전략, 검증 로직을 한 번에 설정합니다.
2단계는 같은 대화 세션에서 내 조건을 입력하는 폼입니다.
동일 단지를 재분석하거나 다른 단지를 비교할 때는 2단계 폼만 바꿔 제출하면 됩니다.

### 프롬프트를 보내기 전에

1단계 프롬프트는 AI의 행동 규칙을 설정하는 역할을 합니다.
숫자 생성 금지, 데이터 부족 시 조건부 종료, 표 먼저 작성 후 결론 도출 등의 규칙이 포함되어 있습니다.
이 프롬프트를 먼저 보내지 않으면 AI가 임의로 수치를 생성하거나 검증 없이 결론을 내릴 수 있습니다.

### 실행 방법

1. ChatGPT(최신모델 권장)에서 새 대화창(New Chat)을 엽니다.
2. 웹 검색 기능을 켭니다. AI가 최신 실거래 데이터를 직접 조회합니다.
3. 아래 1단계 프롬프트를 복사해서 붙여넣습니다.
4. AI가 "사용자 입력 폼"을 출력하고 입력을 기다립니다.

💡 **Tip.** 이 대화창은 2단계 이후에도 계속 사용합니다. 창을 닫지 마세요. 동일 단지 재분석 시에는 "재분석"이라고 요청하면 기존 분석 카드를 참조하여 비교 결과를 보여줍니다.

### 1단계 프롬프트

```
[ROLE]
당신은 한국 아파트 시장의 "시장 사이클 기반 적정가/타이밍" 전문 분석가입니다.
실거래 데이터와 비교단지 분석을 통해 현재 가격의 적정성을 판단하고, 실무적인 매수 액션 플랜을 제시합니다.

[핵심 원칙]
- 숫자는 반드시 출처와 함께 제시하세요 (임의 생성 금지).
- 데이터가 부족하면 "조건부/보류"로 종료하세요.
- 표를 먼저 채운 후에만 결론을 도출하세요.

[2-STEP WORKFLOW | 매우 중요]
- 본 대화는 2단계로 진행됩니다.
  1) (당신) "사용자 입력 폼"을 출력하고, 사용자가 폼을 채워 제출하도록 안내하세요.
  2) (사용자) 폼을 제출하면, 당신은 입력 검증을 통과한 경우에만 웹 수집 → DATA PACK(6개 표) 작성 → 적정가 범위 산출 → 전략 판정 → 실행 트리거 설계를 수행하세요.

[절대 규칙: 입력 완료 전 분석 금지]
- 사용자의 입력이 '필수 항목'을 모두 충족하기 전에는 웹 수집/분석을 시작하지 마세요.
- 누락/모순이 있으면 분석 대신, 최대 6개 이내의 짧은 질문으로 보완 입력을 요청하세요.
- 질문이 6개를 초과할 것 같으면, "가장 중요한 3개"만 먼저 질문하고 나머지는 다음 턴으로 넘기세요.

[필수 항목 (모두 필수)]
- 분석 기준일
- 분석 모드 (갭투자 최적화 / 실거주 기반 매수 타이밍)
- 단지명
- 주소 (시/구/동)
- 면적 입력 (전용㎡, 평형, 타입 중 편한 방식으로 최소 1개 이상)
- 판단 대상 가격 + 가격 성격 (최근 실거래 / 현재 호가 / 희망 매수가)
- 예상 전세금 + 전세 유형 (갭투자 시 필수)
- 투자 목표 (시세차익/현금흐름/혼합) / 보유 기간 / 실거주 계획
- 자금 (자기자본 / 대출여력 / 월상환한도)
- 규제·세금 전제 (1주택/다주택, 규제/비규제지역)
- 리스크 성향 (보수/중립/공격)
- 비교 단지 후보 (3~6개 또는 "없음")

[면적 해석/표준화 규칙(강제)]
- 분석/비교의 최종 기준 면적은 반드시 "전용㎡"로 내부에서 통일하세요.
- 입력 해석 기본값:
  * "59타입/84타입" 등 숫자만 입력되거나 타입으로 입력되면 '전용㎡' 기준으로 간주하세요.
  * "24평/34평" 등 '평'으로 입력되면 '공급면적' 기준으로 간주하세요.
- 데이터 수집 단계에서 해당 단지의 "전용/공급 매핑"을 먼저 확인하고, 전용면적 ±5㎡ 이내는 미세한 차이가 있더라도 모두 동일 평형군(동일 면적군)으로 묶어서 분석하세요.
- 매핑이 복수이거나 불명확하여 오류 가능성이 크면 분석을 중단하고 질문으로 재확인하세요.

[SAFE RULES (필수 준수)]
1. 숫자 생성 금지: 모든 가격에 (출처, 기준월)을 반드시 표기하세요.
2. 데이터 부족 시: "조건부/보류"로 종료하고 체크리스트를 제시하세요.
3. 단위 표준화: 억원(소수 1자리) + 전용㎡당(만원)을 병기하세요.
4. 평형 혼합 금지: 전용면적 ±5㎡ 이내만 동일군으로 처리하세요.
5. 전세 구분 필수: 실거래/시세/호가를 반드시 표시하세요.
6. 표 먼저: 표가 비어 있으면 결론을 내리지 마세요.

[FAIL-FAST (데이터 충분성 체크)]
다음 중 2개 이상 부족하면 "조건부/보류"로 처리하세요.
- 최근 12개월 실거래 5건 이상 (해당 면적군)
- 전세 정보 1개 이상 (갭투자 시 필수)
- 비교단지 3개 이상 실거래 밴드 확보

[MISSION]
지정된 단지의 현재 가격이 적정한지, 지금이 매수 타이밍인지 판단하세요.
1. DATA PACK(6개 표)을 먼저 작성하세요.
2. 적정가 범위를 산출하세요 (3가지 모델).
3. 실무 기반의 매수 전략(즉시 계약/가격 협상/급매 대기/다른 단지 추천/보류)을 판정하세요.
4. 실행 트리거를 설계하세요 (30/60/90일).

[SEARCH STRATEGY]
필수 검색(최소 6회)을 수행하세요.
1. "[단지명] 전용[면적] 실거래가 [현재연월]" - 최신 거래 확인
2. "[단지명] 전세 실거래 [현재연월]" - 전세가율 계산
3. "[지역명] 아파트 시세 동향 [현재연월]" - 시장 사이클 진단
4. "[단지명] 거래량 [현재연도]" - 유동성 판단
5. "[지역명] 입주물량 [현재연도] [다음연도]" - 공급 리스크
6. "[지역명] 금리 영향 부동산 [현재연월]" - 정책 이벤트

비교단지 검색(단지당 2회)을 수행하세요.
- "[비교단지명] 전용[면적] 실거래가" - 밴드 비교
- "[비교단지명] 전세가" - 전세가율 비교

소스 우선순위를 적용하세요.
- A등급: 국토부 실거래가, 한국부동산원
- B등급: KB부동산, 호갱노노, 아실(asil)
- C등급: 네이버부동산, 부동산 뉴스

[OUTPUT FORMAT]
0) 합의 결과 한 줄을 먼저 제시하세요.
> [단지명] [면적]㎡ 적정가 범위: [N]억~[N]억 / 갭투자 가능 여부: [가능·불가] / 최종 액션: [즉시 계약 / 가격 협상 / 급매 대기 / 다른 단지 추천 / 보류]

1) DATA PACK(6개 표)을 작성하세요.
- (표 1) 대상 단지 실거래 + 실거래 밴드 요약
- (표 2) 거래량 (월별)
- (표 3) 전세 정보
- (표 4) 비교단지 실거래 밴드
- (표 5) 공급/정책/금리 이벤트
- (표 6) 자금·레버리지 체크 (갭투자 시 필수)
- 데이터 신뢰도 + 누락 데이터

2) 적정가 범위 산출(3가지 모델)을 제시하세요.
- 모델 A: 실거래 밴드 기반 (25%~75%)
- 모델 B: 전세가율 역산 (55~65% 가정)
- 모델 C: 비교단지 ㎡당 가격 적용
- 통합: 3개 모델 중앙값 산출 및 현재 가격(호가/가계약금)의 위치 평가

3) 최종 액션 플랜 판정 (아래 기준을 참고하여 논리적으로 도출하세요)
- **즉시 계약**: 현재 가격이 적정가 하단에 위치하며, 거래량 증가 및 긍정적 지표가 다수인 경우
- **가격 협상**: 현재 호가가 적정가보다 다소 높으나, 매수자 우위 시장이거나 조정 여지가 있는 경우 (목표 타겟가 제시 필수)
- **급매 대기**: 현재 호가가 고평가되어 있으나, 단지 자체의 가치는 훌륭하여 하락장/공급장 등에 맞춰 모니터링이 필요한 경우
- **다른 단지 추천**: 자금 한도를 초과하거나, 비교 단지 대비 뚜렷하게 고평가되어 투자 메리트가 떨어지는 경우
- **보류**: 데이터 부족, 정책 리스크 과다, 갭투자 자금 펑크 등 치명적 결함이 있는 경우

4) 실행 트리거 설계 (액션별 가이드)를 제시하세요.
- 향후 30일/60일/90일 내 체크해야 할 시장 신호 3가지
- 계약 진행 또는 대기 시 유의할 리스크 체크리스트 5가지

5) Self-criticism TOP5 (분석의 약점 + 검증 방법)를 제시하세요.
```

### AI 응답 확인하기

폼을 제출하면 AI가 필수 항목 검증 후 분석을 시작합니다.
정대리(예시 1: 민락푸르지오 84㎡, 타이밍 매수 모드)의 경우 다음과 같은 결과가 나옵니다.

```
[합의 결과 한 줄]
> 민락푸르지오 84㎡ 적정가 범위: 5.4억~6.2억 / 갭투자 가능 여부: 불가 / 최종 액션: 가격 협상
종합 신뢰도: Med-High
```

이어서 DATA PACK 6개 표가 출력됩니다.

| 모델             | 산출 방식                   | 적정가 범위         | 신뢰도 |
| ---------------- | --------------------------- | ------------------- | ------ |
| **모델 A** | 실거래 밴드 기반 (25%~75%)  | 5.2~6.0억           | High   |
| **모델 B** | 전세가율 역산 (55~65% 가정) | 5.0~5.8억           | Med    |
| **모델 C** | 비교단지 ㎡당 가격 적용     | 5.5~6.2억           | High   |
| **통합**   | 3개 모델 중앙값             | **5.4~6.2억** | -      |

"합의 결과 한 줄"에서 핵심을 빠르게 확인할 수 있습니다.
5.8억 매물이 5.4~6.2억 범위 안에 있으므로 터무니없이 비싼 가격은 아닙니다.
다만 "적정"이 "저렴"을 의미하지는 않습니다.

모델 B의 신뢰도가 Med인 이유는 전세가율 데이터가 부족하거나 가정이 들어갔기 때문입니다.
신뢰도가 Med 이하인 모델의 결과는 참고만 하고, 직접 국토부 실거래가 공개시스템에서 확인해야 합니다.

분석 결과 마지막에는 요약 로그 카드가 출력됩니다.

```
요약 로그 카드
분석ID: 001 | 기준일: 2026-01-05 | 단지명: 민락푸르지오
전용면적: 84㎡ | 적정가 범위: 5.4억~6.2억
현재 가격 위치: 범위 내 (5.8억) | 최종 액션: 가격 협상 | 신뢰도: Med-High
```

이 카드는 같은 단지를 재분석하거나 다른 단지 결과와 비교할 때 기준점이 됩니다.

> ⚠️ **주의:** AI가 응답한 전세가율과 실거래 수치는 반드시 공식 출처와 교차검증해야 합니다. 특히 전세가율은 실거래 기준인지 호가 기준인지에 따라 값이 크게 달라집니다. 국토부 실거래가 공개시스템에서 직접 확인하세요.

### 확인할 포인트

| 확인 항목                            | 기준                                       | 문제 시 대처                                       |
| ------------------------------------ | ------------------------------------------ | -------------------------------------------------- |
| 합의 결과 한 줄이 첫 번째로 나왔는가 | 적정가 범위와 최종 판정이 서두에 있어야 함 | "합의 결과 한 줄을 먼저 출력해 주세요"라고 요청    |
| DATA PACK 표 6개가 모두 채워졌는가   | 빈 셀이 있으면 데이터 부족                 | 분석 기간을 24개월로 확대 요청                     |
| 신뢰도 표시가 있는가                 | High/Med/Low로 표시되어야 함               | 없으면 "각 모델의 신뢰도를 표시해 주세요"라고 요청 |
| 요약 로그 카드가 마지막에 있는가     | 분석 결과 맨 끝에 표 형식으로 있어야 함    | "요약 로그 카드를 출력해 주세요"라고 요청          |

## AI 분석 결과 두 개 이상으로 교차 확인하기

실습 ㉒에서와 마찬가지로, 하나의 AI만 사용하지 말고 두 개 이상의 AI에 같은 프롬프트를 입력해서 결과를 비교하는 것을 권장합니다.

ChatGPT와 Gemini에 동일한 조건을 입력했을 때 적정가 범위가 다를 수 있습니다.

| 비교 항목             | ChatGPT           | Gemini           |
| --------------------- | ----------------- | ---------------- |
| **적정가 범위** | 5.4~6.2억         | 5.2~5.9억        |
| **최종 액션**   | 가격 협상         | 급매 대기        |
| **핵심 이유**   | GTX-C 기대감 반영 | 거래량 회복 고려 |
| **데이터 출처** | aptrank, 국토부   | KB부동산, 네이버 |

두 AI의 결과가 다를 때는 **겹치는 범위(5.4~5.9억)**가 더 신뢰할 수 있는 구간입니다.

- **적정가 범위가 겹치면**: 겹치는 구간을 신뢰 구간으로 활용합니다.
- **적정가 범위가 다르면**: 각 AI가 어떤 데이터와 모델을 기반으로 산출했는지 확인하고, 직접 검증이 필요합니다.
- **액션이 다르면**: 두 결과 중 더 보수적이고 안전한 결과(예: 가격 협상보다는 급매 대기)를 택하는 것이 안전합니다.

## 실행 트리거를 행동 기준으로 만들기

AI 분석 결과에는 매수 트리거와 보류/철회 트리거가 포함됩니다.
이것은 "어떤 조건이 되면 매수하고, 어떤 조건이 되면 대기하라"는 구체적인 행동 기준입니다.
분석 결과를 읽는 것에서 멈추지 말고, 트리거를 메모장에 기록해 두세요.

아래는 민락푸르지오 분석에서 나온 트리거 예시입니다.

**매수 트리거 (이 조건 충족 시 매수 고려)**

| 트리거 | 조건                      | 확인 방법                  |
| ------ | ------------------------- | -------------------------- |
| 가격   | 5.6억 이하 매물이 나오면  | 네이버부동산 매물 모니터링 |
| 거래   | 동일 라인 2~3건 연속 체결 | 국토부 실거래 확인         |
| 호가   | 호가-실거래 괴리 5% 이내  | 아실 데이터 비교           |

**보류/철회 트리거 (이 조건 발생 시 대기)**

| 트리거 | 조건               | 위험도              |
| ------ | ------------------ | ------------------- |
| 가격   | 5.2억 이하로 급락  | 시장 전체 하락 신호 |
| 공급   | GTX-C 연기 발표    | 기대감 소멸         |
| 금리   | 기준금리 추가 인상 | 대출 부담 증가      |

트리거를 기록해두면 감정에 흔들리지 않고 조건에 따라 움직일 수 있습니다.
"가격 협상" 또는 "급매 대기"라는 액션 결과값의 실질적인 의미도 이 트리거 목록이 명확하게 만들어 줍니다.

---

## 핵심 정리

- **시장 사이클이 먼저**: 같은 가격도 사이클 위치에 따라 비쌀 수도, 쌀 수도 있어 가격 숫자만으로는 판단할 수 없습니다.
- **3가지 모델 교차 산출**: 실거래 밴드, 전세가율 역산, 비교단지 ㎡당 가격을 교차 적용하여 적정가 범위의 신뢰도를 높입니다.
- **모드 선택이 핵심**: 갭투자는 전세가율 임계값, 타이밍 매수는 적정가 대비 현재 가격 위치를 우선 확인합니다.
- **트리거로 행동 기준 확보**: AI 분석 결과에서 매수·보류 트리거를 뽑아두면 조건에 따라 움직이는 기준이 생깁니다.
